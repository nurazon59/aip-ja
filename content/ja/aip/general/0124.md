# リソース関連付け

APIには、通常のツリー構造ではきれいに表現できないリソース階層が存在することがある。例えば、リソースが1つだけでなく2つの他のリソースタイプとの多対一の関係を持つ場合がある。あるいは、リソースが別のリソースタイプとの多対多の関係を持つ場合もある。

## ガイダンス

リソースは最大で1つの正規の親を持た**なければならず**（must）、`List`リクエストは動作するために2つの異なる「親」を必要と**してはならない**（must not）。

### 複数の多対一関連付け

リソースが複数のリソースタイプとの多対一の関係を持つ場合、そのうちの最大1つを正規の親として選択**しなければならない**（must）。リソースは、リソース上の他のフィールドを通じて他のリソースと関連付けられ**てもよい**（may）。

```proto
message Book {
  // The resource name pattern for Book indicates that Publisher is the
  // canonical parent.
  option (google.api.resource) = {
    type: "library.googleapis.com/Book"
    pattern: "publishers/{publisher}/books/{book}"
  };

  // The resource name for the book.
  string name = 1 [(google.api.field_behavior) = IDENTIFIER];

  // The resource name for the book's author.
  string author = 2 [(google.api.resource_reference) = {
    type: "library.googleapis.com/Author"
  }];
}
```

この方法で複数の関連付けを持つリソースをリストする場合、RPCは[AIP-132][]で説明されているように、`string parent`フィールドをrequiredとして扱わ**なければならず**（must）、追加の必須引数を追加**してはならない**（must not）。RPCは、ユーザーが他のリソース関連付けでフィルタリングできるようにする`string filter`フィールドを含める**べきである**（should）（[AIP-160][]で説明）。

**注:** リソース参照フィールドは、参照されるリソースの`name`フィールドで使用されるものと同じリソース名形式を受け入れ**なければならない**（must）。

### 多対多関連付け

多対多の関連付けは、APIではリレーショナルデータベースほど一般的ではない。その理由の一部は、ネットワークインターフェース上でモデル化して提示するのがより難しいためである。

APIは多対多の関係を含ん**でもよい**（may）、[AIP-144][]で繰り返しフィールドについて説明されている原則に従い、リソース名のリストを含む繰り返しフィールドを使用**すべきである**（should）。

```proto
message Book {
  option (google.api.resource) = {
    type: "library.googleapis.com/Book"
    pattern: "publishers/{publisher}/books/{book}"
  };

  string name = 1 [(google.api.field_behavior) = IDENTIFIER];

  // The resource names for the book's authors.
  repeated string authors = 2 [(google.api.resource_reference) = {
    type: "library.googleapis.com/Author"
  }];
}
```

**注:** 繰り返しフィールドの詳細（アトミックな変更などの一般的な問題の処理方法を含む）については、[AIP-144][]を参照のこと。

繰り返しフィールドの使用が制約が強すぎる場合、または関連付けとともに追加のメタデータが必要な場合、APIは2つの一対多の関連付けを持つサブリソースを使用して多対多の関係をモデル化**してもよい**（may）。

```proto
message BookAuthor {
  // The resource pattern for BookAuthor indicates that Book is the
  // canonical parent.
  option (google.api.resource) = {
    type: "library.googleapis.com/BookAuthor"
    pattern: "publishers/{publisher}/books/{book}/authors/{book_author}"
  };

  // The resource name for the book-author association.
  string name = 1 [(google.api.field_behavior) = IDENTIFIER];

  // The resource name for the author.
  string author = 2 [(google.api.resource_reference) = {
    type: "library.googleapis.com/Author"
  }];

  // Other fields...
}
```

**注:** サブリソースを使用してリソース間の関連付けをモデル化することは、関係に追加のメタデータが必要な場合、または繰り返しフィールドの使用に関する制限によりそのアプローチの使用が妨げられる場合にのみ推奨される。
