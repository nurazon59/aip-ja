# 標準メソッド: Delete

REST APIでは、リソースを削除するためにリソースのURI（例：`/v1/publishers/{publisher}/books/{book}`）に`DELETE`リクエストを行うのが慣例である。

リソース指向設計（AIP-121）は、`Delete`メソッドを通じてこのパターンを尊重する。これらのRPCはそのリソースを表すURIを受け入れ、通常は空のレスポンスを返す。

## ガイダンス

APIは、ユーザーにとって価値がない場合を除き、一般的にリソースに対してdeleteメソッドを提供**すべきである**（should）。

Deleteメソッドは以下のパターンを使用して指定される：

```proto
rpc DeleteBook(DeleteBookRequest) returns (google.protobuf.Empty) {
  option (google.api.http) = {
    delete: "/v1/{name=publishers/*/books/*}"
  };
  option (google.api.method_signature) = "name";
}
```

- RPC名は`Delete`という単語で始まら**なければならない**（must）。RPC名の残りの部分は、リソースのメッセージ名の単数形**にすべきである**（should）。
- リクエスト メッセージはRPC名と一致し、`Request`サフィックスが付かな**ければならない**（must）。
- レスポンス メッセージは`google.protobuf.Empty`**すべきである**（should）。
  - リソースが[ソフト削除](#soft-delete)される場合、レスポンス メッセージはリソース自体**すべきである**（should）。
  - delete RPCが[long-running](#long-running-delete)の場合、レスポンス メッセージは正しいレスポンスに解決される`google.longrunning.Operation`**でなければならない**（must）。
- HTTP動詞は`DELETE`**でなければならない**（must）。
- リソース名を受け取るリクエスト メッセージフィールドはURIパスにマッピング**すべきである**（should）。
  - このフィールドは`name`**すべきである**（should）。
  - `name`フィールドはURIパス内の唯一の変数**すべきである**（should）。残りのパラメータはすべてURIクエリパラメータにマッピング**すべきである**（should）。
- `google.api.http`アノテーションに`body`キーがあっては**ならない**（must not）。
- `google.api.method_signature`アノテーションは1つだけ存在**すべきであり**（should）、値は`"name"`とする。etagまたはforceフィールドが使用される場合、それらはシグネチャに含め**てもよい**（may）。
- APIが[管理プレーン][management plane]で動作している場合、操作は[強い一貫性][strong consistency]を持つ**べきである**（should）：delete操作の完了は、リソースの存在が定常状態に達し、リソース状態の読み取りが一貫したレスポンスを返すことを意味**しなければならない**（must）。
- 子リソースが存在する場合、APIは`FAILED_PRECONDITION`エラーで失敗**しなければならない**（must）。親リソースと子リソースの削除を強制する必要がある場合は、[カスケード削除](#cascading-delete)のガイダンスを参照のこと。
  - 唯一の子リソース型が[シングルトン][aip-156]である場合、シングルトンのライフサイクルは親リソースのライフサイクルに紐づいているため、削除は許可**されなければならない**（must）。これは、同じ親リソースに複数の異なるシングルトン リソース型がある場合でも適用される。

Deleteメソッドは、リソースが存在し正常に削除された場合にのみ成功**すべきである**（should）。リソースが存在しなかった場合、メソッドは`NOT_FOUND`エラーを送信**すべきである**（should）。

### リクエスト メッセージ

Deleteメソッドは共通のリクエスト メッセージパターンを実装する：

```proto
message DeleteBookRequest {
  // 削除するブックの名前。
  // 形式: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      type: "library.googleapis.com/Book"
    }];
}
```

- `name`フィールドを含め**なければならない**（must）。`name`**とすべきである**（should）。
  - フィールドは [required としてアノテートされるべきである][aip-203]（should）。
  - フィールドは参照する[リソース型][aip-123]を識別**しなければならない**（must）。
- フィールドのコメントはリソースパターンを文書化**すべきである**（should）。
- リクエスト メッセージに他の必須フィールドを含めては**ならない**（must not）、別のAIPで説明されているものを除き、他のオプショナル フィールドを含める**べきではない**（should not）。

### Long-running delete

一部のリソースは、通常のAPIリクエストとして合理的な時間よりも削除に時間がかかる。この場合、APIは代わりにlong-running operationを使用**すべきである**（should）：

```proto
rpc DeleteBook(DeleteBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    delete: "/v1/{name=publishers/*/books/*}"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "OperationMetadata"
  };
}
```

- レスポンス型は、RPCがlong-runningでない場合の適切な戻り値型（ほとんどのDelete RPCでは`google.protobuf.Empty`、ソフト削除の場合はリソース自体（AIP-164））に設定**しなければならない**（must）。
- `response_type`と`metadata_type`の両方を指定**しなければならない**（must）（たとえ`google.protobuf.Empty`であっても）。

### カスケード削除

ユーザーがリソースとそのすべての子リソースを削除できるようにする必要がある場合がある。しかし、削除は通常永続的であるため、ユーザーが誤って削除しないようにすることも重要であり、削除された子リソースの再構築は非常に困難な場合がある。

APIが子リソースを持つ可能性のあるリソースの削除を許可する場合、APIはリクエストに`bool force`フィールドを提供**すべきであり**（should）、ユーザーはこれを設定してカスケード削除に明示的にオプトインする。

```proto
message DeletePublisherRequest {
  // 削除する出版社の名前。
  // 形式: publishers/{publisher}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      type: "library.googleapis.com/Publisher"
    }];

  // trueに設定すると、この出版社のすべてのブックも削除される。
  // （それ以外の場合、リクエストは出版社にブックがない場合にのみ動作する。）
  bool force = 2;
}
```

`force`フィールドが`false`（または未設定）で子リソースが存在する場合、APIは`FAILED_PRECONDITION`エラーで失敗**しなければならない**（must）。

### 保護された削除

ユーザーが、削除されるリソースに変更が加えられていないことを確認する必要がある場合がある。リソースが[etag][]を提供する場合、deleteリクエストはetagを受け入れ**てもよい**（may）（requiredまたはoptionalとして）：

```proto
message DeleteBookRequest {
  // 削除するブックの名前。
  // 形式: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      type: "library.googleapis.com/Book"
    }];

  // オプショナル。ブックのetag。
  // これが提供された場合、サーバーのetagと一致しなければならない。
  string etag = 2;
}
```

etagが提供され、サーバー計算値と一致しない場合、リクエストは`ABORTED`エラーコードで失敗**しなければならない**（must）。

**注:** 宣言的フレンドリーなリソース（AIP-128）は、Deleteリクエストに`etag`フィールドを提供**しなければならない**（must）。

### 存在すれば削除

サービスがクライアント割り当てのリソース名を使用する場合、`Delete`メソッドは`bool allow_missing`フィールドを公開**してもよい**（may）。これにより、ユーザーが存在しないリソースを削除しようとした場合にメソッドが成功する（その場合、リクエストはno-opとなる）：

```proto
message DeleteBookRequest {
  // 削除するブック。
  // 形式: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];

  // trueに設定され、ブックが見つからない場合、リクエストは成功するが
  // サーバー上では何も実行されない。
  bool allow_missing = 2;
}
```

より具体的には、`allow_missing`フラグは以下の動作をトリガーする：

- メソッド呼び出しが存在しないリソースに対するものである場合、リクエストはno-opとなる。
  - `etag`フィールドは無視される。
- メソッド呼び出しが既に存在するリソースに対するものである場合、リソースは削除される（他のチェックを受ける）。

**注:** 宣言的フレンドリーなリソース（AIP-128）は`bool allow_missing`フィールドを公開**すべきである**（should）。

### エラー

ユーザーがリソースにアクセスする権限を持っていない場合（リソースが存在するかどうかに関わらず）、サービスは`PERMISSION_DENIED`（HTTP 403）でエラーに**しなければならない**（must）。権限はリソースの存在を確認する**前に**チェック**されなければならない**（must）。

ユーザーが適切な権限を持っているが、リクエストされたリソースが存在しない場合、サービスは`allow_missing`が`true`に設定されていない限り、`NOT_FOUND`（HTTP 404）でエラーに**しなければならない**（must）。
