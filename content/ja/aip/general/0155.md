# リクエスト識別

APIによっては、特定のリクエストに対して一意の顧客提供識別子を持つことが有用な場合がある。これは、並列プロセスからのリクエストの重複排除、リトライの安全性の確保、監査など、いくつかの目的に役立つ。

リクエストIDの最も重要な目的は、冪等性の保証を提供することである。つまり、同じリクエストを複数回発行しても、後続の呼び出しが影響を及ぼさないようにする。ネットワーク障害が発生した場合、クライアントはリクエストを再試行し、サーバーは重複を検出してリクエストが1回だけ処理されることを保証できる。

## ガイダンス

APIは、特定のリクエストを一意に識別するために、リクエストメッセージ（標準メソッドのものを含む）に `string request_id` パラメータを追加 **してもよい**（may）。

```proto
message CreateBookRequest {
  // この書籍が作成される親リソース。
  // 形式: publishers/{publisher}
  string parent = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      child_type: "library.googleapis.com/Book"
    }];

  // 書籍に使用するID。書籍のリソース名の最後の構成要素となる。
  //
  // この値は4〜63文字とし、有効な文字は /[a-z][0-9]-/ とする。
  string book_id = 2 [(google.api.field_behavior) = REQUIRED];

  // 作成する書籍。
  Book book = 3 [(google.api.field_behavior) = REQUIRED];

  // このリクエストの一意識別子。36 ASCII文字に制限される。
  // ランダムUUIDが推奨される。
  // このリクエストは、`request_id` が提供された場合にのみ冪等である。
  string request_id = 4 [(google.api.field_info).format = UUID4];
}
```

- リクエストIDの提供は、冪等性を保証 **しなければならない**（must）。
  - 重複リクエストが検出された場合、サーバーは以前に成功したリクエストのレスポンスを返す **すべきである**（should）。なぜなら、クライアントは以前のレスポンスを受信していない可能性が高いためである。
  - APIは、リクエストIDを有効とする任意の合理的な期間を選択 **してもよい**（may）。
- `request_id` フィールドは、それが適用されるリクエストメッセージ上に提供され**なければならない**（must）（リソース自体のフィールドであっては**ならない**（must not））。
- リクエストIDはオプション **すべきである**（should）。
- リクエストIDはUUID**であるべきである**（should）であり、UUIDのみを有効な形式として許可 **してもよい**（may）。リクエストIDの形式制約は文書化 **しなければならない**（must）。
  - UUIDであるリクエストIDは、`(google.api.field_info).format = UUID4` のように拡張機能を使用して `google.api.FieldInfo.Format` 値 `UUID4` でアノテーションされ**なければならない**（must）。詳細は [AIP-202](./0202.md) を参照。

### 古くなった成功レスポンス

まれな状況では、同一の成功レスポンスを返すことが不可能な場合がある。例えば、リソースを作成する重複リクエストが、リソースが作成されただけでなく、その後更新された後に到着する場合がある。サービスには履歴データを保持する他の必要がないため、同一の成功レスポンスを返すことはもはや現実的ではない。

この状況では、メソッドは代わりにリソースの現在の状態を返して **してもよい**（may）。言い換えれば、過去の成功レスポンスを、より最新のデータを反映した同様のレスポンスに置き換えることは許容される。

## 理由

### リクエスト識別へのUUIDの使用

値が一意である必要がある場合、形式を自由にするとAPIコンシューマーが誤って重複する識別子を提供する可能性がある。そのため、普遍的に一意な識別子に標準化することで、適切に実装された場合に衝突の可能性を大幅に削減できる。
