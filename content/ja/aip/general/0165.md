# 条件ベース削除

APIは、削除するリソースの個別のリソース名を必要とするのではなく、フィルタパラメータのセットに基づいて大量のリソースを削除するメカニズムを提供する必要がある場合がある。

これは稀なケースであり、ユーザーが一度に数千以上のリソースを削除する必要がある状況のために予約されている。その場合、通常のBatch Deleteパターン（[AIP-235][]）は扱いにくく不便になる。

## ガイダンス

**重要：** ほとんどのAPIは、リソースの削除にはDelete（[AIP-135][]）またはBatch Delete（[AIP-235][]）のみを使用する**すべきであり**（should）、条件に基づく削除を実装**すべきではない**（should not）。これは、削除が一般的に不可逆的であり、この種の操作ではユーザーが誤って大量のデータを失いやすいためである。

APIは、フィルタ文字列に基づいて大量のリソースを削除することを許可するPurgeメソッドを実装**してもよい**（may）。ただし、これはBatch Delete（[AIP-235][]）パターンでは目的の目標を達成するのに不十分な場合にのみ行う**すべきである**（should）：

```proto
rpc PurgeBooks(PurgeBooksRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{parent=publishers/*}/books:purge"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "PurgeBooksResponse"
    metadata_type: "PurgeBooksMetadata"
  };
}
```

- RPCの名前は単語`Purge`で始め**なければならない**（must）。RPC名の残りの部分は、削除するリソースの複数形にする**すべきである**（should）。
- リクエストメッセージはRPC名と一致し、`Request`サフィックスを付け**なければならない**（must）。
- レスポンスタイプは、RPC名と一致し`Response`サフィックスが付いたメッセージに解決される`google.longrunning.Operation`（AIP-151参照）で**なければならない**（must）。
- HTTP動詞は`POST`で**なければならず**（must）、`body`は`"*"`で**なければならない**（must）。
- URIパスはリソースのコレクションを表す**すべきである**（should）。
- `parent`フィールドはURIに含める**すべきである**（should）。APIが複数の親にわたる削除をサポートしたい場合、[AIP-159][]に従って`-`文字を受け入れる**すべきである**（should）。

### リクエストメッセージ

Purgeメソッドは共通のリクエストメッセージパターンを実装する：

```proto
message PurgeBooksRequest {
  // ブックをパージするパブリッシャー。
  // パブリッシャーをまたいでブックをパージするには、"publishers/-"を送信する。
  string parent = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      child_type: "library.googleapis.com/Book"
    }];

  // パージするブックに一致するフィルター。
  string filter = 2 [(google.api.field_behavior) = REQUIRED];

  // 実際にパージを実行する。
  // `force`がfalseに設定されている場合、削除されるリソース名のサンプルを返す。
  bool force = 3;
}
```

- 単一の`string parent`フィールドを含める**すべきである**（should）（リソースがトップレベルでない限り）。
  - フィールドは[requiredとしてアノテーションされる][aip-203]**すべきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する**すべきである**（should）。
- 単一の`string filter`フィールドを含め**なければならず**（must）、Listメソッド（AIP-160）と同じセマンティクスに従う**しなければならない**（must）。
  - [requiredとしてアノテーションされる][aip-203]**すべきである**（should）。
  - ワイルドカード値`"*"`は、すべてを削除するためにサポート**してもよい**（may）。
- 単一の`bool force`フィールドを含め**なければならない**（must）。設定されていない場合、APIは実際に削除を実行せずに、削除されるリソースの数とそれらのリソースのサンプルを返さ**なければならない**（must）。

### レスポンスメッセージ

Purgeメソッドは共通のレスポンスメッセージパターンを実装する：

```proto
message PurgeBooksResponse {
  // このリクエストで削除されたブックの数（または`force`がfalseの場合は削除されるブックの数）。
  int32 purge_count = 1;

  // 削除されるブックのリソース名のサンプル。
  // `force`がfalseに設定されている場合のみ設定される。
  repeated string purge_sample = 2 [(google.api.resource_reference) = {
    type: "library.googleapis.com/Book"
  }];
}
```

- 単一の`int32 purge_count`フィールドを含める**すべきであり**（should）、削除された（または削除される）リソースの数を提供する。このカウントはAIP-158の`total_size`と同様に推定値**してもよい**（may）（ただし、サービスはその場合は文書化**すべきである**（should））。
- `repeated string purge_sample`フィールドを含める**すべきである**（should）：`force`が`false`の場合、削除されるリソース名のサンプルを提供する**すべきである**（should）。`force`がtrueの場合、このフィールドは設定**すべきではない**（should not）。
  - サンプルは明らかな誤りを捕捉するのに十分なサイズにする**すべきである**（should）：目安として100が適切。APIはサイズを文書化**すべきであり**（should）、また最大値であることを文書化**すべきである**（should）（より少ない数が送信される可能性がある）。
  - サンプルはランダム**してもよく**（may）、または決定論的**してもよい**（may）（最初に一致したリソース名など）。APIはどのアプローチを使用するかを文書化**すべきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する**すべきである**（should）。

**注：** `purge_count`と`purge_sample`が含まれていなくても、`force`フィールドはリクエストに含め**なければならない**（must）。
