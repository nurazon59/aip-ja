# ポリシープレビュー

ポリシーとは、他のリソースへのアクセスを許可または拒否するルールを提供するリソースである。一般に、ポリシーの結果は特定の結果セットに評価できる。

適切な検証なしにポリシーを変更すると、顧客のインフラストラクチャ全体に深刻な影響を与える可能性のある意図しない結果を引き起こす可能性がある。リソースを安全に更新するには、ポリシーロールアウトAPIを介してこれらの変更をテストすることが有益である。

プレビューはポリシーリソースのロールアウト安全メカニズムであり、顧客が変更を本番稼働させる前に、本番トラフィックに対する提案変更の影響を検証する機能を提供する。トラフィックに対するポリシー評価の結果はログに記録され、顧客は変更の正確性をテストするために必要なデータを得る。

ファイアウォールポリシーは、プレビューに適したケースの例である。新しい構成をトラフィックに対して評価し、どのIPが許可または拒否されるかを観察できる。これにより、顧客は提案された変更を本番に昇格させるかどうかの判断を導くデータを得る。

ポリシーのプレビューの期待されるフローは以下の通り：

1. ユーザーは、本番ポリシーを置き換えることを意図した新しいポリシー構成を含む実験を作成する。
2. ユーザーは"startPreview"メソッドを使用して、本番トラフィックに対する本番ポリシーと実験ポリシーの評価を比較するログの生成を開始する。
3. ユーザーはログを検査して、実験が意図した結果をもたらすかどうかを判断する。
4. ユーザーは"commit"メソッドを使用して、実験を本番に昇格させる。

## ガイダンス

### 非目標

この提案は、ポリシーロールアウトのための安全メカニズムのみを対象とする。ポリシー以外のリソースの安全なロールアウトは対象外である。

### 実験

プレビューされるポリシーの新しい構成は、ポリシーの下のネストされたコレクションとして保存される。これらのネストされたコレクションは実験として知られている。

仮想的なポリシーリソースである`Policy`が全体を通して使用される。以下のリソース名パターンを持つ：

`projects/{project}/locations/{location}/policies/{policy}`

プレビューまたはその他の安全なロールアウトプラクティスに使用されるリソースの実験バージョンは、新しいリソースタイプを使用して`Policy`の下のネストされたコレクションとして表現される。リソースタイプは*RegularResourceType*`Experiment`という命名規則に従う**しなければならない**（must）。

実験コレクションには以下のパターンが使用される：

`projects/{project}/locations/{location}/policies/{policy}/experiments/{experiment}`

実験を表すprotoは以下を含む**しなければならない**（must）：

1. `name`や`etag`などのリソースの必須トップレベルフィールド
2. テストされているポリシーメッセージ自体
3. 特定のリソースタイプの実験のプレビューに関するメタデータを含むフィールド`preview_metadata`

```proto
message PolicyExperiment {

  // google.api.resource, name, and other annotations and fields

  // The policy experiment. This Policy will be used to preview the effects of
  // the change but will not affect live traffic.
  Policy policy = 2;

  // The metadata associated with this policy experiment.
  PolicyPreviewMetadata preview_metadata = 3
      [(google.api.field_behavior) = OUTPUT_ONLY];

  // Allows clients to store small amounts of arbitrary data.
  map<string, string> annotations = 4;
}
```

- 実験のprotoは、本番ポリシーと同じ型のトップレベルフィールドを持た**しなければならない**（must）。
  - 本番リソースタイプとして命名**しなければならない**（must）。例えば、実験がFirewallPolicy用の場合、このフィールドは`firewall_policy`と命名**しなければならない**（must）。
  - 埋め込まれた`policy`メッセージ内の名前は、本番ポリシーの名前で**しなければならない**（must）。
- ユーザーが実験を昇格させる準備ができたら、`policy`メッセージを本番ポリシーにコピーし、実験を削除**しなければならない**（must）。これは手動で、または"commit"カスタム メソッドを介して行うことができる。
- プロダクトは、単一の本番ポリシーに対して複数の実験を同時にプレビューすることをサポート**してもよい**（may）。
  - 各実験は、各エントリの前にlog_prefixを付けたログを生成し、ユーザーが実験の結果を本番ポリシーの動作と比較できるようにしなければならない。
  - 特定の本番ポリシーの実験構成の数は、特定の数に上限が設定されても**してもよい**（may）。上限は文書化**しなければならない**（must）。
- カスケード削除は発生**しなければならない**（must）：本番ポリシーが削除された場合、すべての実験も削除**しなければならない**（must）。
- `map<string,string>` [annotations][aip-128-annotations]は、クライアントが少量の任意のデータを保存できるように**しなければならない**（must）。

### メタデータ

`preview_metadata`は、実験のプレビューのすべてのメタデータを追跡する。メッセージは*RegularResourceType*`PreviewMetadata`という規則に従う**しなければならない**（must）。これは、同じサービス内で実験を持つリソースタイプごとにprotoを一意に定義できるようにするためである。

```proto
message PolicyPreviewMetadata {
  // Possible values of the state of previewing the experiment.
  enum State {
    // Default value. This value is unused.
    STATE_UNDEFINED = 0;

    // The experiment is actively previewing.
    ACTIVE = 1;

    // The previewing of the experiment has been stopped.
    SUSPENDED = 2;
  }

  // The state of previewing the experiment.
  State state = 1;

  // An identifying string common to all logs generated when previewing the
  // experiment. Searching all logs for this string will isolate the results.
  string log_prefix = 2;

  // The most recent time at which this experiment started previewing.
  google.protobuf.Timestamp start_time = 3;

  // The most recent time at which this experiment stopped previewing.
  google.protobuf.Timestamp stop_time = 4;
}
```

- `PolicyPreviewMetadata`は、上記のprotoで定義されたフィールドを持た**しなければならない**（must）。
  - サービスまたはリソースが必要とする場合、追加のフィールドを持っても**してもよい**（may）。
- 実験が最初にプレビューされるとき、`preview_metadata`は存在してはならない**しなければならない**（must）。
  - "startPreview"メソッドが使用された後、実験上に存在する。
- すべての`preview_metadata`フィールドはOutput onlyで**しなければならない**（must）。
- `state`は、プレビューが開始または停止されると`ACTIVE`と`SUSPENDED`の間で変化する。これは、それぞれ"startPreview"または"stopPreview"カスタム メソッドが呼び出されたときに発生する。
- "startPreview"カスタム メソッドが初めて使用されるとき、システムは`preview_metadata`を作成し、以下を実行**しなければならない**（must）：
  - `state`を`ACTIVE`に設定**しなければならない**（must）。
  - `start_time`を現在時刻で設定**しなければならない**（must）。
    - `start_time`は`state`が`ACTIVE`に変更されるたびに更新**しなければならない**（must）。
  - システム生成の`log_prefix`文字列を設定**しなければならない**（must）。これはシステム開発者によってハードコードされた事前定義定数である。
  - 同じ値が、特定のリソースタイプの実験のプレビューに使用される。例えば、FirewallPolicyの場合は"FirewallPolicyPreviewLog"である。
- "stopPreview"カスタム メソッドが使用されるとき、システムは以下を実行**しなければならない**（must）：
  - `state`を`SUSPENDED`に設定**しなければならない**（must）。
  - `stop_time`を現在時刻で設定**しなければならない**（must）。

### メソッド

#### create

- リソースはlong-running [Create][aip-133-long-running]を使用して作成**しなければならない**（must）。`google.longrunning.operation_info.response_type`は`PolicyExperiment`で**しなければならない**（must）。
- プレビューする新しい実験の作成は、以下のユースケースをサポート**しなければならない**（must）：
  - 新しいポリシーのプレビュー。
  - 既存の本番ポリシーへの更新のプレビュー。
  - 現在のポリシーの削除のプレビュー。
- 更新および削除のユースケースでは、実験内の`policy`フィールドは、名前を含む本番ポリシーの完全なペイロードがコピーされていなければ**しなければならない**（must）。
  - ユーザーは、更新をプレビューするためにルールを新しい意図された状態に設定**しなければならない**（must）。
  - ユーザーは、削除をプレビューするためにルールをno-opを表すように設定**しなければならない**（must）。
- 新しいポリシーをプレビューするには、システムは以下を実行する必要がある：
  - システムが本番ポリシーなしのネストされたコレクションをサポートしない場合、ユーザーは本番ポリシーを作成し、ルールをno-opに設定**しなければならない**（must）。例えば、no-opポリシーのルールは空でも**してもよい**（may）。
    - 実験はno-opポリシーの子として作成される。
- システムが本番ポリシーに対して複数の実験のプレビューをサポートする場合、"create"を複数回呼び出すと、複数の実験が作成**しなければならない**（must）。

#### update

- リソースはlong-running [Update][aip-134-long-running]を使用して更新**しなければならない**（must）。`google.longrunning.operation_info.response_type`は`PolicyExperiment`で**しなければならない**（must）。
- `policy`内の名前は変更して**してはならない**（must not）が、他のフィールドはプレビューされる実験を変更するために変更できる。この`policy`は本番ポリシーを置き換えることを意図しており、本番ポリシーの名前は変更して**してはならない**（must not）。
- システムは、更新時に`state`が`ACTIVE`だった場合、`state`を`SUSPENDED`に設定**しなければならない**（must）。
  - これは、ユーザーがプレビューされている実験の異なるバージョンを容易に区別できるようにするためである。

#### get
- 標準メソッド[Get][aip-131]は、`PolicyExperiment`リソースタイプに含める**しなければならない**（must）。

#### list

- 標準メソッド[List][aip-132]は、`PolicyExperiment`リソースタイプに含める**しなければならない**（must）。
- `PolicyPreviewMetadata`でのフィルタリングは、どの実験がアクティブにプレビューされているかを示す。
  - 例えば、以下のフィルタ文字列はプレビュー中の実験を含むListレスポンスを返す：preview_metadata.state = ACTIVE。

#### delete

- リソースはlong-running [Delete][aip-135-long-running]を使用して削除**しなければならない**（must）。`google.longrunning.operation_info.response_type`は`PolicyExperiment`で**しなければならない**（must）。

#### startPreview

```proto
// Starts previewing a PolicyExperiment. This triggers the system to start
// generating logs to evaluate the PolicyExperiment.
rpc StartPreviewPolicyExperiment(StartPreviewPolicyExperimentRequest)
    returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=policies/*/experiments/*}:startPreview"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "PolicyExperiment"
    metadata_type: "StartPreviewPolicyExperimentMetadata"
  };
}

// The request message for the startPreview custom method.
message StartPreviewPolicyExperimentRequest {
  // The name of the PolicyExperiment.
  string name = 1;
}
```

- このカスタム メソッドは必須である。
- `google.longrunning.Operation.metadata_type`は[Long-running operations][aip-151]のガイダンスに従う**しなければならない**（must）。
- このメソッドは、システムが実験をプレビューするためのログの生成を開始するようにトリガー**しなければならない**（must）。
- メソッドが正常に呼び出されるたびに、システムは`PolicyPreviewMetadata`内の以下の値を設定**しなければならない**（must）：
  - `log_prefix`を事前定義定数に。
  - `start_time`を現在時刻に。
  - `state`を`ACTIVE`に。
- no-opを表すルールを持つ実験に対してメソッドが呼び出された場合、システムは本番ポリシーの削除をプレビュー**しなければならない**（must）。

#### stopPreview

```proto
// Stops previewing a PolicyExperiment. This triggers the system to stop
// generating logs to evaluate the PolicyExperiment.
rpc StopPreviewPolicyExperiment(StopPreviewPolicyExperimentRequest)
    returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=policies/*/experiments/*}:stopPreview"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "PolicyExperiment"
    metadata_type: "StopPreviewPolicyExperimentMetadata"
  };
}

// The request message for the stopPreview custom method.
message StopPreviewPolicyExperimentRequest {
  // The name of the PolicyExperiment.
  string name = 1;
}
```

- このカスタム メソッドは必須である。
- `google.longrunning.Operation.metadata_type`は[Long-running operations][aip-151]のガイダンスに従う**しなければならない**（must）。
- このメソッドは、システムが実験をプレビューするためのログの生成を停止するようにトリガー**しなければならない**（must）。
- メソッドが正常に呼び出されるたびに、システムは`PolicyPreviewMetadata`内の以下の値を設定**しなければならない**（must）：
  - `stop_time`を現在時刻に。
  - `state`を`SUSPENDED`に。

#### commit

リソースは、実験を昇格させるために"commit"と呼ばれる新しいカスタム メソッドを公開**してもよい**（may）。システムは実験から`policy`を本番ポリシーにコピーし、その後実験を削除する。

宣言的クライアントは、必要に応じて"commit"を呼び出す代わりに、実験から本番ポリシーに手動でフィールドをコピーし、その後実験を削除しても**してもよい**（may）。

```proto
// Commits a PolicyExperiment. This copies the PolicyExperiment's policy message
// to the live policy then deletes the PolicyExperiment.
rpc CommitPolicyExperiment(CommitPolicyExperimentRequest)
    returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=policies/*/experiments/*}:commit"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "CommitPolicyExperimentMetadata"
  };
}

// The request message for the commit custom method.
message CommitPolicyExperimentRequest {
  string name = 1;
  string etag = 2;
  string parent_etag = 3;
}
```

- `google.longrunning.Operation.metadata_type`は[Long-running operations][aip-151]のガイダンスに従う**しなければならない**（must）。
- メソッドは、実験から`policy`を本番ポリシーにアトミックにコピーし、その後実験を削除**しなければならない**（must）。
- 実験が"commit"に失敗した場合、そのプレビューは停止して**してはならない**（must not）。本番ポリシーは更新して**してはならない**（must not）。
- メソッドは任意のステートの実験に対して呼び出すことができる。
- commitが成功するためには、`etag`が実験のものと一致**しなければならない**（must）。これは、ユーザーが意図しないバージョンの実験をコミットしないようにするためである。
  - `etag`が提供されない場合、APIは成功して**してはならない**（must not）。これは、ユーザーが意図せずに異なるバージョンの実験をコミットすることを防ぐためである。
  - `parent_etag`を提供して、実験が本番ポリシーの特定のバージョンを上書きすることを保証しても**してもよい**（may）。
- このメソッドは冪等ではなく、同じ実験に対して2回呼び出すと、最初の呼び出しで実験が削除されるため、404 NOT_FOUNDを返す**しなければならない**（must）。

### 本番ポリシーAPIメソッドの変更

#### delete

- 本番ポリシーの削除は、すべての実験を削除**しなければならない**（must）。
- 実験を維持しながら本番ポリシーの効果を無効にするには、このメソッドを使用する代わりに、本番ポリシーをno-opポリシーに変更**しなければならない**（must）。

### ロギング

ロギングは、ユーザーが実験を本番に昇格させるべきかどうかを評価するために重要である。

ログには、評価された実験の結果、その実験に関連付けられた`etag`と本番ポリシーのそれ、および`log_prefix`の値が含まれ**しなければならない**（must）。
  - `etag`フィールドは、ユーザーが本番構成と実験構成のどちらがログで評価されているかを識別するのに役立つ。
  - `log_prefix`は、ユーザーが実験のプレビュー用に特に生成されたログを他のユースケースから分離するのに役立つ。

全体として、これらのログはユーザーが実験を本番に昇格させるかどうかの決定を下すのに役立つ。
