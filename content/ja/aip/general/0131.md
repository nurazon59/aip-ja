# 標準メソッド: Get

REST APIでは、リソースを取得するためにリソースのURI（例：`/v1/publishers/{publisher}/books/{book}`）に`GET`リクエストを行うのが慣例である。

リソース指向設計（[AIP-121][]）は、`Get`メソッドを通じてこのパターンを尊重する。これらのRPCはそのリソースを表すURIを受け入れ、リソースを返す。

## ガイダンス

APIはリソースに対してgetメソッドを提供**しなければならない**（must）。getメソッドの目的は、単一のリソースからデータを返すことである。

Getメソッドは以下のパターンを使用して指定される：

```proto
rpc GetBook(GetBookRequest) returns (Book) {
  option (google.api.http) = {
    get: "/v1/{name=publishers/*/books/*}"
  };
  option (google.api.method_signature) = "name";
}
```

- RPC名は`Get`という単語で始まら**なければならない**（must）。RPC名の残りの部分は、リソースのメッセージ名の単数形**とすべきである**（should）。
- リクエスト メッセージはRPC名と一致し、`Request`サフィックスが付かな**ければならない**（must）。
- レスポンス メッセージはリソース自体**でなければならない**（must）。（`GetBookResponse`は存在しない。）
  - レスポンスは通常、完全に設定されたリソースを含む**べきである**（should）。ただし、部分的なレスポンスを返す理由がある場合を除く（[AIP-157][]参照）。
- HTTP動詞は`GET`**でなければならない**（must）。
- URIにはリソース名に対応する単一の変数フィールドが含まれる**べきである**（should）。
  - このフィールドは`name`**とすべきである**（should）。
  - URIにはこのフィールドに対応する変数がある**べきである**（should）。
  - `name`フィールドはURIパス内の唯一の変数**とすべきである**（should）。残りのパラメータはすべてURIクエリパラメータにマッピング**すべきである**（should）。
- `google.api.http`アノテーションに`body`キーがあっては**ならない**（must not）。
- `google.api.method_signature`アノテーションは1つだけ存在**すべきである**（should）。値は`"name"`とする。

### リクエスト メッセージ

Getメソッドは共通のリクエスト メッセージパターンを実装する：

```proto
message GetBookRequest {
  // 取得するブックの名前。
  // 形式: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      type: "library.googleapis.com/Book"
    }];
}
```

- リソース名フィールドを含め**なければならない**（must）。`name`**とすべきである**（should）。
  - フィールドは [required としてアノテートされる][aip-203]**べきである**（should）。
  - フィールドは参照する[リソース型][aip-123]を識別**しなければならない**（must）。
- `name`フィールドのコメントはリソースパターンを文書化**すべきである**（should）。
- リクエスト メッセージに他の必須フィールドを含めては**ならない**（must not）、別のAIPで説明されているものを除き、他のオプショナル フィールドを含める**べきではない**（should not）。

**注:** リクエストオブジェクトの`name`フィールドは、RPCの`google.api.http`アノテーションの`name`変数に対応する。これにより、REST/JSONインターフェース使用時に、リクエストの`name`フィールドがURLの値に基づいて設定される。
