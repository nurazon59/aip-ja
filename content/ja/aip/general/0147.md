# 機密フィールド

APIは、基礎となるサービスによって保存されることを意図しているが、データの機密性のため書き込み後に読み取られることを意図していない、秘密の暗号鍵などの機密情報を収集する必要がある場合がある。この種のデータについては、APIのリクエストおよびレスポンスにおける機密データの表現に特別な考慮が必要である。

## ガイダンス

機密情報がリソース全体の存在に必須である場合、データは対応する出力フィールドなしの[input-onlyフィールド][input-only]として受け付ける**すべきである**（should）。機密データはリソースが存在するために必須であるため、APIのユーザーはリソースの存在が機密データの保存を意味すると想定してもよい。例えば：

```proto
message SelfManagedKeypair {
  string name = 1 [(google.api.field_behavior) = IDENTIFIER];

  // The public key data in PEM-encoded form.
  bytes public_key = 2;

  // The private key data in PEM-encoded form.
  bytes private_key = 3 [
    (google.api.field_behavior) = INPUT_ONLY];
}
```

機密情報が包含リソース内でオプションである場合、機密情報が存在するかどうかを示すために、`_set`という接尾辞を持つ[output-only][]のブール型フィールドを使用**すべきである**（should）。例えば：

```proto
message Integration {
  string name = 1 [(google.api.field_behavior) = IDENTIFIER];
  string uri = 2;

  // A secret to be passed in the `Authorization` header of the webhook.
  string shared_secret = 3 [
    (google.api.field_behavior) = INPUT_ONLY];

  // True if a `shared_secret` has been set for this Integration.
  bool shared_secret_set = 4 [
    (google.api.field_behavior) = OUTPUT_ONLY];
}
```

機密情報を完全に読み返すことを許可せずに識別できるようにすることが重要な場合、ブール型の`_set`フィールドの代わりに、`obfuscated_`プレフィックスを付けた同じ型のフィールドを使用**してもよく**（may）、機密情報に関するコンテキスト情報を提供できる。難読化の具体的な性質はこのAIPの範囲外である。例えば：

```proto
message AccountRecoverySettings {
  // An email to use for account recovery.
  string email = 1 [
    (google.api.field_behavior) = INPUT_ONLY];

  // An obfuscated representation of the recovery email. For example,
  // `ada@example.com` might be represented as `a**@e*****e.com`.
  string obfuscated_email = 2 [
    (google.api.field_behavior) = OUTPUT_ONLY];
}
```
