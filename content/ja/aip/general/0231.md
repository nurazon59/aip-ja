# バッチメソッド: Get

一貫した時点（例：読み取りトランザクションの使用）でリソースの特定のセットを取得できるようにする必要があるAPIがある。バッチGetメソッドはこの機能を提供する。

## ガイダンス

APIは以下のパターンを使用してBatch Getをサポート**してもよい**（may）：

```proto
rpc BatchGetBooks(BatchGetBooksRequest) returns (BatchGetBooksResponse) {
  option (google.api.http) = {
    get: "/v1/{parent=publishers/*}/books:batchGet"
  };
}
```

- RPC名は`BatchGet`で始まる**しなければならない**（must）。RPC名の残りの部分は、取得されるリソースの複数形にする**すべきである**（should）。
- リクエストメッセージとレスポンスメッセージは、RPC名に`Request`および`Response`サフィックスを付けたものに一致**しなければならない**（must）。
- HTTP動詞は`GET`**しなければならない**（must）。
- HTTP URIは`:batchGet`で終わる**しなければならない**（must）。
- URIパスは、単純なCRUD操作で使用されるコレクションと一致するリソースのコレクションを表す**すべきである**（should）。操作が複数の親にまたがる場合、ダッシュ（`-`）をワイルドカードとして受け入れても**してもよい**（may）。
- `google.api.http`アノテーションにbodyキーがあって**はならない**（must not）。
- 操作はアトミックで**しなければならない**（must）：すべてのリソースに対して失敗するか、すべてのリソースに対して成功するかのいずれかでなければならない（部分的な成功は不可）。部分的な失敗が必要な状況では、`List`（[AIP-132][]）メソッドを使用**すべきである**（should）。
  - 操作が複数のロケーションにまたがり、少なくとも1つのロケーションがダウンしている場合、操作は失敗**しなければならない**（must）。

### リクエストメッセージ

バッチGetメソッドのリクエストは、以下のパターンで指定**すべきである**（should）：

```proto
message BatchGetBooksRequest {
  // The parent resource shared by all books being retrieved.
  // Format: publishers/{publisher}
  // If this is set, the parent of all of the books specified in `names`
  // must match this field.
  string parent = 1 [
    (google.api.resource_reference) = {
      child_type: "library.googleapis.com/Book"
    }];

  // The names of the books to retrieve.
  // A maximum of 1000 books can be retrieved in a batch.
  // Format: publishers/{publisher}/books/{book}
  repeated string names = 2 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference) = {
      type: "library.googleapis.com/Book"
    }];
}
```

- 取得されるリソースがトップレベルリソースでない限り、`parent`フィールドを含める**すべきである**（should）。これによりURIへの包含と単一の権限チェックが容易になる。呼び出し元がこのフィールドを設定し、取得されるリソースの名前の親コレクションが一致しない場合、リクエストは失敗**しなければならない**（must）。
  - このフィールドは、リクエストごとに1つの親のみが許可される場合、requiredにする**すべきである**（should）。
  - フィールドは、それが参照する[リソースタイプ][aip-122-parent]を識別**すべきである**（should）。
  - フィールドのコメントはリソースパターンを文書化**すべきである**（should）。
- リクエストメッセージには、取得するリソースを指定するリソース名を受け入れるrepeatedフィールドを含める**しなければならない**（must）。フィールドは`names`と命名**すべきである**（should）。
  - リソース名が提供されない場合、APIは`INVALID_ARGUMENT`でエラーを返す**すべきである**（should）。
  - フィールドはrequiredにする**すべきである**（should）。
  - フィールドは、それが参照する[リソースタイプ][aip-122-names]を識別**すべきである**（should）。
  - フィールドのコメントはリソースパターンを文書化**すべきである**（should）。
- `name`以外のフィールドは、[標準Getリクエスト][request-message]から「hoist」しても**してもよい**（may）。これらのフィールドがリソースごとに異なる値を受け入れるようにする方法はない。これが必要な場合は、[代替リクエストメッセージ形式](#nested-request-objects)を使用すること。
- バッチGetはページネーションをサポート**すべきではない**（should not）。なぜなら、API呼び出間でのトランザクション性の実装や強制が極めて困難であり、リクエストがとにかくレスポンスの正確な範囲を定義するためである。
- リクエストメッセージには、その他のrequiredフィールドを含めて**してはならず**（must not）、本AIPまたは他のAIPで説明されているものを除き、他のoptionalフィールドを含める**すべきではない**（should not）。
- `names`フィールドの上のコメントは、許可されるリクエストの最大数を文書化**すべきである**（should）。

### レスポンスメッセージ

バッチGetメソッドのレスポンスは、以下のパターンで指定**すべきである**（should）：

```proto
message BatchGetBooksResponse {
  // Books requested.
  repeated Book books = 1;
}
```

- レスポンスメッセージには、取得されるリソースに対応する1つのrepeatedフィールドを含める**しなければならない**（must）。
- レスポンス内のブックの順序は、リクエスト内の名前と同じで**しなければならない**（must）。

### ネストされたリクエストオブジェクト

[標準Getリクエストメッセージ][request-message]に、リソース名以外でリクエストされるリソース間で異なる必要があるフィールドが含まれている場合、バッチメッセージは代わりに[標準Getリクエストメッセージ][request-message]の`repeated`フィールドを保持しても**してもよい**（may）。これは、ユースケースで本当に必要でない限り、一般的には推奨されない。

このアプローチを使用するバッチGetメソッドのリクエストは、以下のパターンで指定**すべきである**（should）：

```proto
message BatchGetBooksRequest {
  // The parent resource shared by all books being retrieved.
  // Format: publishers/{publisher}
  // If this is set, the parent field in the GetBookRequest messages
  // must either be empty or match this field.
  string parent = 1 [
    (google.api.resource_reference) = {
      child_type: "library.googleapis.com/Book"
    }];

  // The requests specifying the books to retrieve.
  // A maximum of 1000 books can be retrieved in a batch.
  repeated GetBookRequest requests = 2
    [(google.api.field_behavior) = REQUIRED];
}
```

- `parent`フィールドを含める**すべきである**（should）。呼び出し元がこのフィールドを設定し、取得されるリソースの名前の親コレクションが一致しない場合、リクエストは失敗**しなければならない**（must）。
  - このフィールドは、リクエストごとに1つの親のみが許可される場合、requiredにする**すべきである**（should）。
  - フィールドは、それが参照する[リソースタイプ][aip-122-parent]を識別**すべきである**（should）。
  - フィールドのコメントはリソースパターンを文書化**すべきである**（should）。
- リクエストメッセージには、[標準Getメソッド][request-message]で指定されているように、取得するリソースを指定するリクエストメッセージを受け入れるrepeatedフィールドを含める**しなければならない**（must）。フィールドは`requests`と命名**すべきである**（should）。
  - フィールドはrequiredにする**すべきである**（should）。
- 他のフィールドは[標準Getリクエスト][request-message]から「hoist」しても**してもよい**（may）。これは、フィールドがバッチレベルまたは子リクエストレベルのいずれかで設定できることを意味する。`parent`と同様に、バッチレベルと子リクエストレベルの両方で同じフィールドに値が設定された場合、値は一致**しなければならない**（must）。
- バッチGetはページネーションをサポート**すべきではない**（should not）。なぜなら、API呼び出間でのトランザクション性の実装や強制が極めて困難であり、リクエストがとにかくレスポンスの正確な範囲を定義するためである。
- リクエストメッセージには、その他のrequiredフィールドを含めて**してはならず**（must not）、本AIPまたは他のAIPで説明されているものを除き、他のoptionalフィールドを含める**すべきではない**（should not）。
- `requests`フィールドの上のコメントは、許可されるリクエストの最大数を文書化**すべきである**（should）。
