# ソフトデリート

クライアントがソフトデリートと復元（undelete）機能を求める理由はいくつかあるが、最も重要な理由はミスからの回復である。復元をサポートするサービスは、ユーザーが誤って削除したリソースを回復することを可能にする。

## ガイダンス

API は、ユーザーが誤ってリソースを削除し、回復する必要がある状況に対応するために、「復元」（undelete）機能をサポート**してもよい**（may）。

リソースが復元をサポートする必要がある場合、`Delete` メソッドは、リソースを削除済みとしてマークするにとどめ、システムから完全に削除しないようにし**なければならない**（must）。メソッドがこのように動作する場合、`google.protobuf.Empty` ではなく更新されたリソースを返す**すべきである**（should）。

ソフトデリートをサポートするリソースは、AIP-148 で説明されている `delete_time` と `purge_time` の両方のフィールドを持つ**すべきである**（should）。さらに、リソースに `state` フィールド（AIP-216）が含まれている場合、リソースは `DELETED` 状態値を含む**すべきである**（should）。

### 復元（Undelete）

ソフトデリートをサポートするリソースは、`Undelete` メソッドを提供する**すべきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
}
```

- HTTP 動詞は `POST` で**なければならない**（must）。
- `body` 句は `"*"` で**なければならない**（must）。
- レスポンスメッセージはリソースそのもので**なければならない**（must）。`UndeleteBookResponse` は存在しない。
  - 実行不可能な場合を除き、レスポンスには完全に設定されたリソースを含める**すべきである**（should）。
  - 復元 RPC が[長時間実行](#長時間実行の復元)である場合、レスポンスメッセージは `google.longrunning.Operation` で**なければならず**（must）、これはリソースそのものに解決される。

### 復元リクエストメッセージ

復元メソッドは共通のリクエストメッセージパターンを実装する：

```proto
message UndeleteBookRequest {
  // The name of the deleted book.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"];
}
```

- `name` フィールドを含め**なければならない**（must）。これは `name` と呼ぶ**すべきである**（should）。
  - フィールドは [required としてアノテーション][aip-203] する**すべきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する**すべきである**（should）。
  - フィールドのコメントはリソースパターンを文書化する**すべきである**（should）。
- リクエストメッセージには、この AIP または他の AIP で説明されているものを除き、他の必須フィールドを含めては**ならず**（must not）、他のオプションフィールドを含める**べきではない**（should not）。

### 長時間実行の復元

一部のリソースでは、復元に通常の API リクエストとして妥当な時間よりも長くかかる場合がある。この状況では、API は代わりに長時間実行オペレーション（AIP-151）を使用する**すべきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "Book"
    metadata_type: "OperationMetadata"
  };
}
```

- レスポンスタイプはリソースに設定し**なければならない**（must）（RPC が長時間実行でない場合の戻り値の型となるもの）。
- `response_type` と `metadata_type` の両方のフィールドを指定し**なければならない**（must）。

### 完全削除（Expunge）

ソフトデリートをサポートするリソースは、ユーザーがリソースの即時完全削除をトリガーできるようにする `Expunge` カスタム メソッドを提供**してもよい**（may）。このメソッドは、現在 `CREATING`、`READY`、または `SOFT_DELETED` 状態（例: `delete_time` が設定されている）にあるリソースに対して操作できる。

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.protobuf.Empty) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.api.method_signature) = "name";
}
```

- URI は `:expunge` サフィックスを持つカスタム メソッドを使用する。
- HTTP 動詞は POST とし、body 句は `"*"` とする。
- レスポンスメッセージは `google.protobuf.Empty` または `google.longrunning.Operation` とする。

### 長時間実行の完全削除

完全削除プロセスにかなりの時間がかかる場合、メソッドは代わりに `google.longrunning.Operation`（AIP-151）と**してもよい**（may）：

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "OperationMetadata"
  };
  option (google.api.method_signature) = "name";
}
```

### 完全削除リクエストメッセージ

完全削除メソッドは共通のリクエストメッセージパターンを実装する：

```proto
message ExpungeBookRequest {
  // The name of the soft-deleted book to expunge.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];
}
```

- リクエストメッセージは完全削除するリソースを名前で参照し**なければならない**（must）。
- 他のリクエストフィールドを含める**べきではない**（should not）。

### List と Get

ソフトデリートされたリソースは、デフォルトでは `List`（AIP-132）のレスポンスに含める**べきではない**（should not）（`bool show_deleted` が true の場合を除く）。ソフトデリートされたリソースに対する `Get`（AIP-131）リクエストは、リソースを返す**すべきである**（should）（`NOT_FOUND` エラーではなく）。

ソフトデリートを行う API は、合理的な期間（30 日など）後の自動完全削除、ユーザーによる有効期限の設定（AIP-214）、またはリソースの無期限保持など、リソースを完全削除するための適切な戦略を選択**してもよい**（may）。どの戦略が選択された場合でも、API はソフトデリートされたリソースがいつ完全に削除されるかを文書化する**すべきである**（should）。

### 宣言的クライアント向けリソース

ソフトデリート可能なリソースは、宣言的クライアントにおいてハードデリートされたリソースよりも劣るエクスペリエンスとなる。ソフトデリートされたリソースの ID は、カスタム メソッド（復元）が呼び出されない限り再利用できないため、命令型クライアントを導入するか、カスタム メソッドの使用を組み込むために手書きのコードが必要になる。

### エラー

リソースが存在するかどうかに関わらず、ユーザーがリソースにアクセスする権限を持たない場合、サービスは `PERMISSION_DENIED`（HTTP 403）でエラーとし**なければならない**（must）。権限は、リソースが存在するかどうかを確認する前にチェックし**なければならない**（must）。

ユーザーが適切な権限を持っているが、リクエストされたリソースが存在しない（作成されたことがない、またはすでに完全削除されている）場合、サービスは `NOT_FOUND`（HTTP 404）でエラーとし**なければならない**（must）。

ソフト `Delete` を呼び出すユーザーが適切な権限を持っているが、リクエストされたリソースがすでに削除されている場合、`allow_missing` が `true` であればサービスは成功し**なければならず**（must）、`allow_missing` が `false` であれば `NOT_FOUND`（HTTP 404）でエラーとする**すべきである**（should）。

`Undelete` を呼び出すユーザーが適切な権限を持っているが、リクエストされたリソースが削除されていない場合、サービスは `ALREADY_EXISTS`（HTTP 409）で応答し**なければならない**（must）。

`Expunge` を呼び出すユーザーが、存在しない（作成されたことがない、またはすでに完全削除されている）リソースをリクエストした場合、メソッドは `NOT_FOUND`（HTTP 404）を返さ**なければならない**（must）。

リソースが存在するが、ready またはソフトデリート状態にない場合、メソッドは `FAILED_PRECONDITION`（HTTP 400）を返さ**なければならない**（must）。

標準的な権限エラー（`PERMISSION_DENIED`）が適用される。サービスは、標準の削除権限とは別の明示的な完全削除権限を要求し**なければならない**（must）（例: `<service>.<resource>.expunge`）。
