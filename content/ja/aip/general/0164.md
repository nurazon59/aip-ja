# ソフトデリート

クライアントがソフトデリートおよびアンデリート機能を求める理由はいくつかあるが、最も重要な理由の一つはミスからの回復である。アンデリートをサポートするサービスは、ユーザーが誤って削除したリソースを復元することを可能にする。

## ガイダンス

APIは、ユーザーが誤ってリソースを削除し、復元する必要がある状況に対応するために、アンデリート機能を**してもよい**（may）。

リソースがアンデリートをサポートする必要がある場合、`Delete` メソッドはリソースをシステムから完全に削除するのではなく、単に削除済みとしてマークし**なければならない**（must）。このように動作する場合、`google.protobuf.Empty` ではなく更新されたリソースを返す**べきである**（should）。

ソフトデリートをサポートするリソースは、AIP-148 で説明されている `delete_time` と `purge_time` の両方のフィールドを持つ**べきである**（should）。さらに、リソースに `state` フィールドが含まれる場合（AIP-216）、リソースは `DELETED` 状態値を含む**べきである**（should）。

### アンデリート

ソフトデリートをサポートするリソースは、`Undelete` メソッドを提供する**べきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
}
```

- HTTP メソッドは `POST` で**なければならない**（must）。
- `body` 句は `"*"` で**なければならない**（must）。
- レスポンスメッセージはリソース自体で**なければならない**（must）。`UndeleteBookResponse` は存在しない。
  - 実行不可能でない限り、レスポンスには完全に設定されたリソースを含める**べきである**（should）。
  - アンデリート RPC が[長時間実行](#long-running-undelete)の場合、レスポンスメッセージはリソース自体に解決される `google.longrunning.Operation` で**なければならない**（must）。

### アンデリートリクエストメッセージ

アンデリートメソッドは共通のリクエストメッセージパターンを実装する：

```proto
message UndeleteBookRequest {
  // The name of the deleted book.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"];
}
```

- `name` フィールドを含め**なければならない**（must）。これは `name` という名前にする**べきである**（should）。
  - フィールドは[必須としてアノテーション][aip-203]する**べきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する**べきである**（should）。
  - フィールドのコメントにはリソースパターンを文書化する**べきである**（should）。
- リクエストメッセージには他の必須フィールドを含めて**はならない**（must not）。また、この AIP または他の AIP で説明されているものを除き、他のオプションフィールドを含める**べきではない**（should not）。

### 長時間実行アンデリート

一部のリソースでは、通常の API リクエストとして妥当な時間よりもアンデリートに時間がかかる場合がある。このような状況では、API は代わりに長時間実行オペレーション（AIP-151）を使用する**べきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "Book"
    metadata_type: "OperationMetadata"
  };
}
```

- レスポンスタイプはリソースに設定し**なければならない**（must）（RPC が長時間実行でない場合の戻り値の型）。
- `response_type` と `metadata_type` の両方のフィールドを指定し**なければならない**（must）。

### Expunge

ソフトデリートをサポートするリソースは、`Expunge` カスタムメソッドを提供し、ユーザーが ready 状態またはソフトデリート状態のリソースの即時完全削除をトリガーできるように**してもよい**（may）。このメソッドは、現在 ready 状態またはソフトデリート状態（すなわち `delete_time` が設定されている）のリソースに対して操作できる。

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.protobuf.Empty) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.api.method_signature) = "name";
}
```

- URI は `:expunge` サフィックスを持つカスタムメソッドを使用しなければならない。
- HTTP メソッドは POST でなければならず、`body` 句は `"*"` でなければならない。
- レスポンスメッセージは `google.protobuf.Empty` または `google.longrunning.Operation` でなければならない。

### 長時間実行 Expunge

Expunge プロセスにかなりの時間がかかる場合、代わりに `google.longrunning.Operation`（AIP-151）に**してもよい**（may）：

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "OperationMetadata"
  };
  option (google.api.method_signature) = "name";
}
```

### Expunge リクエストメッセージ

Expunge メソッドは共通のリクエストメッセージパターンを実装する：

```proto
message ExpungeBookRequest {
  // The name of the soft-deleted book to expunge.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];
}
```

- リクエストメッセージは、expunge するリソースを名前で参照し**なければならない**（must）。
- 他のリクエストフィールドは含める**べきではない**（should not）。

### List と Get

ソフトデリートされたリソースは、デフォルトでは `List`（AIP-132）のレスポンスに含める**べきではない**（should not）（`bool show_deleted` が true の場合を除く）。ソフトデリートされたリソースに対する `Get`（AIP-131）リクエストは、（`NOT_FOUND` エラーではなく）リソースを返す**べきである**（should）。

ソフトデリートを行う API は、妥当な期間（30 日など）経過後の自動パージ、ユーザーによる有効期限の設定（AIP-214）、またはリソースの無期限保持など、それらのリソースをパージするための合理的な戦略を選択し**てもよい**（may）。どの戦略を選択する場合でも、API はソフトデリートされたリソースがいつ完全に削除されるかを文書化する**べきである**（should）。

### 宣言的クライアント対応リソース

ソフトデリート可能なリソースは、宣言的クライアントにおいてハードデリートされたリソースよりも劣るエクスペリエンスを持つ。ソフトデリートされたリソースの ID は、カスタムメソッド（アンデリート）が呼び出されない限り再利用できないため、命令的クライアントを導入するか、カスタムメソッドの使用を組み込むために手書きのコードが必要になる。

### エラー

ユーザーがリソースにアクセスする権限を持っていない場合、そのリソースが存在するかどうかに関わらず、サービスは `PERMISSION_DENIED`（HTTP 403）でエラーとし**なければならない**（must）。権限チェックは、リソースが存在するかどうかのチェックよりも先に実行し**なければならない**（must）。

ユーザーが適切な権限を持っているが、要求されたリソースが存在しない場合（一度も作成されていないか、すでに expunge されている）、サービスは `NOT_FOUND`（HTTP 404）でエラーとし**なければならない**（must）。

ソフト `Delete` を呼び出すユーザーが適切な権限を持っているが、要求されたリソースがすでに削除されている場合、`allow_missing` が `true` であればサービスは成功し**なければならない**（must）。`allow_missing` が `false` であれば `NOT_FOUND`（HTTP 404）でエラーとする**べきである**（should）。

`Undelete` を呼び出すユーザーが適切な権限を持っているが、要求されたリソースが削除されていない場合、サービスは `ALREADY_EXISTS`（HTTP 409）で応答し**なければならない**（must）。

`Expunge` を呼び出すユーザーが、存在しないリソース（一度も作成されていないか、すでに expunge されている）を要求した場合、メソッドは `NOT_FOUND`（HTTP 404）を返さ**なければならない**（must）。

リソースが存在するが ready 状態またはソフトデリート状態でない場合、メソッドは `FAILED_PRECONDITION`（HTTP 400）を返さ**なければならない**（must）。

標準的な権限エラー（`PERMISSION_DENIED`）が適用される。サービスは、標準的な削除権限とは別の明示的な expunge 権限（例：`<service>.<resource>.expunge`）を要求し**なければならない**（must）。
