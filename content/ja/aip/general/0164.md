# ソフトデリート

クライアントがソフトデリートと復元（undelete）機能を求める理由はいくつかあるが、最も重要な理由はミスからの回復である。復元をサポートするサービスは、ユーザーが誤って削除したリソースを復元することを可能にする。

## ガイダンス

APIは、ユーザーが誤ってリソースを削除し、復元する必要が生じた場合に対応するため、復元（「undelete」）機能をサポート**してもよい**（may）。

リソースが復元をサポートする必要がある場合、`Delete` メソッドは、リソースを完全にシステムから削除するのではなく、削除済みとしてマークするだけに**しなければならない**（must）。メソッドがこのように動作する場合、`google.protobuf.Empty` ではなく、更新されたリソースを返す**べきである**（should）。

ソフトデリートをサポートするリソースは、AIP-148 に記載されている `delete_time` と `purge_time` フィールドの両方を持つ**べきである**（should）。さらに、リソースに `state` フィールドが含まれる場合（AIP-216）、`DELETED` 状態値を含める**べきである**（should）。

### 復元（Undelete）

ソフトデリートをサポートするリソースは、`Undelete` メソッドを提供する**べきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
}
```

- HTTPメソッドは `POST` に**しなければならない**（must）。
- `body` 句は `"*"` に**しなければならない**（must）。
- レスポンスメッセージはリソースそのものに**しなければならない**（must）。`UndeleteBookResponse` は存在しない。
  - 実現不可能な場合を除き、レスポンスには完全にデータが投入されたリソースを含める**べきである**（should）。
  - 復元 RPC が[長時間実行](#長時間実行の復元undelete)される場合、レスポンスメッセージはリソースそのものに解決される `google.longrunning.Operation` に**しなければならない**（must）。

### 復元リクエストメッセージ

復元メソッドは、共通のリクエストメッセージパターンを実装する：

```proto
message UndeleteBookRequest {
  // The name of the deleted book.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"];
}
```

- `name` フィールドを含め**なければならない**（must）。フィールド名は `name` とすることが推奨される（**should**）。
  - フィールドは [必須としてアノテーション][aip-203] する**べきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を特定する**べきである**（should）。
  - フィールドのコメントにはリソースパターンを記載する**べきである**（should）。
- リクエストメッセージには、他の必須フィールドを含めて**はならない**（must not）。また、本AIPまたは他のAIPに記載されているものを除き、他のオプショナルフィールドを含める**べきではない**（should not）。

### 長時間実行の復元（Undelete）

一部のリソースでは、復元に通常のAPIリクエストとしては妥当でないほどの時間がかかる場合がある。そのような場合、APIは代わりに長時間実行オペレーション（AIP-151）を使用する**べきである**（should）：

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "Book"
    metadata_type: "OperationMetadata"
  };
}
```

- レスポンスタイプはリソース（RPCが長時間実行でない場合の戻り値の型）に設定**しなければならない**（must）。
- `response_type` と `metadata_type` の両方のフィールドを指定**しなければならない**（must）。

### 完全削除（Expunge）

ソフトデリートをサポートするリソースは、ユーザーがリソースの即時完全削除をトリガーできるようにする `Expunge` カスタムメソッドを提供**してもよい**（may）。このメソッドは、現在 `CREATING`、`READY`、または `SOFT_DELETED` 状態にあるリソースに対して動作できる（例：`delete_time` が設定されている場合）。

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.protobuf.Empty) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.api.method_signature) = "name";
}
```

- URI は `:expunge` サフィックスを持つカスタムメソッドを使用しなければならない。
- HTTPメソッドは POST にしなければならず、`body` 句は `"*"` にしなければならない。
- レスポンスメッセージは `google.protobuf.Empty` または `google.longrunning.Operation` にしなければならない。

### 長時間実行の完全削除（Expunge）

完全削除プロセスに大きな時間がかかる場合、メソッドは代わりに長時間実行オペレーション（AIP-151）に**してもよい**（may）：

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "OperationMetadata"
  };
  option (google.api.method_signature) = "name";
}
```

### 完全削除リクエストメッセージ

完全削除メソッドは、共通のリクエストメッセージパターンを実装する：

```proto
message ExpungeBookRequest {
  // The name of the soft-deleted book to expunge.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];
}
```

- リクエストメッセージは、完全削除するリソースを名前で参照**しなければならない**（must）。
- 他のリクエストフィールドは含める**べきではない**（should not）。

### List と Get

ソフトデリートされたリソースは、デフォルトでは `List`（AIP-132）のレスポンスに返される**べきではない**（should not）（`bool show_deleted` が true の場合を除く）。ソフトデリートされたリソースに対する `Get`（AIP-131）リクエストは、リソースを返す**べきである**（should）（`NOT_FOUND` エラーではなく）。

リソースをソフトデリートするAPIは、それらのリソースを完全に削除するための合理的な戦略を選択**してもよい**（may）。これには、合理的な期間（例：30日）後の自動完全削除、ユーザーによる有効期限の設定（AIP-214）、またはリソースの無期限保持が含まれる。選択された戦略に関わらず、APIはソフトデリートされたリソースがいつ完全に削除されるかを文書化する**べきである**（should）。

### 宣言的クライアント対応リソース

ソフトデリート可能なリソースは、ハードデリートされたリソースよりも宣言的クライアントでの使用体験が劣る。ソフトデリートされたリソースのIDは、カスタムメソッド（undelete）が呼び出されない限り再利用できないため、命令的クライアントを導入するか、カスタムメソッドの使用を組み込むための手書きコードが必要になる。

### エラー

ユーザーがリソースにアクセスする権限を持たない場合、リソースの存在有無に関わらず、サービスは `PERMISSION_DENIED`（HTTP 403）エラーを返さ**なければならない**（must）。権限はリソースの存在確認より前にチェックされ**なければならない**（must）。

ユーザーが適切な権限を持っていても、リクエストされたリソースが存在しない（作成されていない、または既に完全削除されている）場合、サービスは `NOT_FOUND`（HTTP 404）エラーを返さ**なければならない**（must）。

適切な権限を持つユーザーがソフト `Delete` を呼び出したが、リクエストされたリソースが既に削除済みの場合、サービスは `allow_missing` が `true` であれば成功**しなければならない**（must）。また、`allow_missing` が `false` の場合は `NOT_FOUND`（HTTP 404）エラーを返す**べきである**（should）。

適切な権限を持つユーザーが `Undelete` を呼び出したが、リクエストされたリソースが削除済みでない場合、サービスは `ALREADY_EXISTS`（HTTP 409）を返さ**なければならない**（must）。

存在しない（作成されていない、または既に完全削除されている）リソースに対してユーザーが `Expunge` をリクエストした場合、メソッドは `NOT_FOUND`（HTTP 404）を返さ**なければならない**（must）。

リソースが存在するが、`READY` または `SOFT_DELETED` 状態でない場合、メソッドは `FAILED_PRECONDITION`（HTTP 400）を返さ**なければならない**（must）。

標準的な権限エラー（`PERMISSION_DENIED`）が適用される。サービスは、標準の削除権限とは別の明示的な完全削除権限（例：`<service>.<resource>.expunge`）を要求**しなければならない**（must）。
