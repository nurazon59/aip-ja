# ソフトデリート

クライアントがソフトデリートおよび復元（undelete）機能を望む理由はいくつかあるが、
その中でも際立つ根源的な理由がひとつある。それはミスからの回復である。
復元をサポートするサービスは、誤って削除されたリソースをユーザーが回復することを
可能にする。

## ガイダンス

APIは、ユーザーが誤ってリソースを削除し、回復する必要がある状況に対応するため、
「復元（undelete）」機能をサポート**してもよい**（may）。

リソースが復元をサポートする必要がある場合、`Delete`メソッドは、リソースを
システムから完全に削除せず、単に削除済みとしてマークするのみで**なければならない**（must）。
このように振る舞う場合、メソッドは`google.protobuf.Empty`ではなく、
更新されたリソースを返す**べきである**（should）。

ソフトデリートをサポートするリソースは、AIP-148で説明されているように、
`delete_time`フィールドと`purge_time`フィールドの両方を持つ**べきである**（should）。
さらに、リソースに`state`フィールドが含まれる場合（AIP-216）、リソースは
`DELETED`状態値を含む**べきである**（should）。

### 復元（Undelete）

ソフトデリートをサポートするリソースは、`Undelete`メソッドを提供する
**べきである**（should）。

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
}
```

- HTTP動詞は`POST`で**なければならない**（must）。
- `body`句は`"*"`で**なければならない**（must）。
- レスポンスメッセージはリソースそのもので**なければならない**（must）。
  `UndeleteBookResponse`は存在しない。
  - レスポンスは、実行不可能な場合を除き、完全に設定されたリソースを含む
    **べきである**（should）。
  - 復元RPCが[長時間実行](#long-running-undelete)である場合、レスポンス
    メッセージはリソースそのものに解決される`google.longrunning.Operation`で
    **なければならない**（must）。

### 復元リクエストメッセージ

復元メソッドは、共通のリクエストメッセージパターンを実装する。

```proto
message UndeleteBookRequest {
  // The name of the deleted book.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"];
}
```

- `name`フィールドを含め**なければならない**（must）。これは`name`と呼ばれる
  **べきである**（should）。
  - フィールドは[必須としてアノテーション][aip-203]される**べきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する
    **べきである**（should）。
  - フィールドのコメントはリソースパターンを文書化する**べきである**（should）。
- リクエストメッセージは、このAIPまたは他のAIPで説明されているものを除き、
  他の必須フィールドを含めては**ならない**（must not）。また、他のオプショナル
  フィールドを含める**べきではない**（should not）。

### 長時間実行復元

一部のリソースでは、復元に通常のAPIリクエストとしては妥当でない時間が
かかる場合がある。このような状況では、APIは代わりに長時間実行オペレーション
（AIP-151）を使用する**べきである**（should）。

```proto
rpc UndeleteBook(UndeleteBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:undelete"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "Book"
    metadata_type: "OperationMetadata"
  };
}
```

- レスポンスタイプは、RPCが長時間実行でなかった場合の戻り値の型である
  リソースに設定され**なければならない**（must）。
- `response_type`と`metadata_type`の両方のフィールドが指定され
  **なければならない**（must）。

### 完全削除（Expunge）

ソフトデリートをサポートするリソースは、ユーザーが準備完了状態または
ソフトデリート状態（例：`delete_time`が設定されている）のリソースの
即時完全削除をトリガーできるようにする`Expunge`カスタムメソッドを
提供**してもよい**（may）。

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.protobuf.Empty) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.api.method_signature) = "name";
}
```

- URIは`:expunge`サフィックスを持つカスタムメソッドを使用し**なければならない**（must）。
- HTTP動詞は`POST`で、`body`句は`"*"`で**なければならない**（must）。
- レスポンスメッセージは`google.protobuf.Empty`または
  `google.longrunning.Operation`で**なければならない**（must）。

### 長時間実行完全削除

完全削除プロセスにかなりの時間がかかる場合、メソッドは代わりに
`google.longrunning.Operation`（AIP-151）であっても**よい**（may）。

```proto
// Permanently deletes a soft-deleted Book.
rpc ExpungeBook(ExpungeBookRequest) returns (google.longrunning.Operation) {
  option (google.api.http) = {
    post: "/v1/{name=publishers/*/books/*}:expunge"
    body: "*"
  };
  option (google.longrunning.operation_info) = {
    response_type: "google.protobuf.Empty"
    metadata_type: "OperationMetadata"
  };
  option (google.api.method_signature) = "name";
}
```

### 完全削除リクエストメッセージ

完全削除メソッドは、共通のリクエストメッセージパターンを実装する。

```proto
message ExpungeBookRequest {
  // The name of the soft-deleted book to expunge.
  // Format: publishers/{publisher}/books/{book}
  string name = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];
}
```

- リクエストメッセージは、完全削除するリソースを名前で参照し
  **なければならない**（must）。
- 他のリクエストフィールドは存在す**べきではない**（should not）。

### List と Get

ソフトデリートされたリソースは、デフォルトでは（`bool show_deleted`が
trueでない限り）`List`（AIP-132）のレスポンスに含める
**べきではない**（should not）。ソフトデリートされたリソースに対する
`Get`（AIP-131）リクエストは、リソースを返す**べきである**（should）
（`NOT_FOUND`エラーではなく）。

リソースをソフトデリートするAPIは、妥当な期間（30日など）経過後の自動パージ、
ユーザーによる有効期限（AIP-214）の設定許可、またはリソースの無期限保持など、
リーズナブルなパージ戦略を選択**してもよい**（may）。選択された戦略にかかわらず、
APIはソフトデリートされたリソースがいつ完全に削除されるかを文書化する
**べきである**（should）。

### 宣言的フレンドリーなリソース

ソフトデリート可能なリソースは、ハードデリートされたリソースと比較して
宣言的クライアントでの体験が劣る。ソフトデリートされたリソースのIDは、
カスタムメソッド（undelete）が呼び出されない限り再利用できないため、
命令的クライアントを導入するか、カスタムメソッドの使用を組み込む
手書きのコードが必要となる。

### エラー

ユーザーがリソースにアクセスする権限を持っていない場合、そのリソースが
存在するかどうかにかかわらず、サービスは`PERMISSION_DENIED`（HTTP 403）で
エラーにし**なければならない**（must）。権限は、リソースの存在確認よりも
先にチェックされ**なければならない**（must）。

ユーザーが適切な権限を持っているが、要求されたリソースが存在しない
（作成されたことがない、またはすでに完全削除されている）場合、サービスは
`NOT_FOUND`（HTTP 404）でエラーにし**なければならない**（must）。

ソフト`Delete`を呼び出すユーザーが適切な権限を持っているが、要求された
リソースがすでに削除されている場合、サービスは`allow_missing`が`true`で
あれば成功し**なければならない**（must）。`allow_missing`が`false`であれば
`NOT_FOUND`（HTTP 404）でエラーにする**べきである**（should）。

`Undelete`を呼び出すユーザーが適切な権限を持っているが、要求された
リソースが削除されていない場合、サービスは`ALREADY_EXISTS`（HTTP 409）で
応答し**なければならない**（must）。

`Expunge`を呼び出すユーザーが存在しない（作成されたことがない、または
すでに完全削除されている）リソースを要求した場合、メソッドは
`NOT_FOUND`（HTTP 404）を返さ**なければならない**（must）。

リソースが存在するが、準備完了状態でもソフトデリート状態でもない場合、
メソッドは`FAILED_PRECONDITION`（HTTP 400）を返さ**なければならない**（must）。

標準的な権限エラー（`PERMISSION_DENIED`）が適用される。サービスは、
標準的な削除権限とは別の明示的な完全削除権限を要求し
**なければならない**（must）（例：`<service>.<resource>.expunge`）。
