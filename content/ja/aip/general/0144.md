# 繰り返しフィールド

APIでデータのリストを表現することは、見た目よりも難しい。ユーザーはリストをその場で変更する必要があることが多く、単一リソース内の長いデータ系列はページネーションにとって課題となる。

## ガイダンス

リソースは、必要に応じてrepeatedフィールドを使用**してもよい**（may）。

```proto
message Book {
  option (google.api.resource) = {
    type: "library.googleapis.com/Book"
    pattern: "publishers/{publisher}/books/{book}"
  };

  string name = 1 [(google.api.field_behavior) = IDENTIFIER];

  repeated string authors = 2;
}
```

- Repeatedフィールドは複数形のフィールド名を使用**しなければならない**（must）。
  - 英語の単数形と複数形が同一の場合（"moose"、"info"）、新しい複数形を作り出すのではなく、辞書に記載されている語を使用**しなければならない**（must）。
- Repeatedフィールドは、単一のリソースペイロードが大きくなりすぎないように、強制的な上限を設ける**べきである**（should）。目安としては100要素が妥当である。
  - Repeatedデータが大きくなりすぎる可能性がある場合、APIは代わりにサブリソースを使用**すべきである**（should）。
- Repeatedフィールドは、別のリソースの本体をインラインで表現しては**ならない**（must not）。代わりに、メッセージは関連するリソースのリソース名を提供する**べきである**（should）。

### スカラーとメッセージ

Repeatedフィールドは、将来追加データが必要にならないことが確実な場合はスカラー型（`string`など）を使用**すべきである**（should）。メッセージ型を使用すると認知的オーバーヘッドが大幅に増加し、コードが複雑になるためである。

ただし、将来追加データが必要になる可能性が高い場合は、repeatedフィールドは並列repeatedフィールドを避けるために、事前にスカラーではなくメッセージを使用**すべきである**（should）。

### 更新戦略

リソースは、repeatedフィールドの更新を可能にするために2つの戦略を使用**してもよい**（may）：標準の[`Update`][aip-134]メソッドを使用した直接更新、またはカスタムの`Add`および`Remove`メソッド。

標準の`Update`メソッドには1つの重要な制限がある：ユーザーはリスト*全体*のみを更新できる。フィールドマスクはrepeatedフィールド内の個々のエントリを扱うことができない。つまり、ユーザーはリソースを読み取り、必要に応じてrepeatedフィールド値を変更し、送り返す必要がある。これは多くの状況、特にrepeatedフィールドのサイズが小さい（10程度未満）と予想され、競合状態が問題にならないか、[ETags][aip-154]で保護できる場合には問題ない。

**注:** 宣言的フレンドリーなリソースは、標準の`Update`メソッドを使用**しなければならず**（must）、`Add`および`Remove`メソッドを導入してはならない。宣言的ツールが特定の関係だけを考慮して他の関係を無視する必要がある場合は、代わりにサブリソースの使用を検討すること。

アトミックな変更が必要な場合、APIは動詞`Add`および`Remove`を使用してカスタムメソッドを定義する**べきである**（should）：

**注:** これらの戦略の両方が制限的すぎる場合は、代わりにサブリソースの使用を検討すること。

```proto
rpc AddAuthor(AddAuthorRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{book=publishers/*/books/*}:addAuthor"
    body: "*"
  };
}

rpc RemoveAuthor(RemoveAuthorRequest) returns (Book) {
  option (google.api.http) = {
    post: "/v1/{book=publishers/*/books/*}:removeAuthor"
    body: "*"
  };
}
```

- 追加または削除されるデータはプリミティブ（通常は`string`）である**べきである**（should）。
  - 主キーを持つより複雑なデータ構造の場合、APIは代わりに`Update`メソッドとともにマップを使用**すべきである**（should）。
- RPC名は単語`Add`または`Remove`で始まら**なければならない**（must）。RPC名の残りの部分は、追加されるフィールドの単数形と**すべきである**（should）。
- リクエストメッセージはRPC名と一致し、`Request`接尾辞が付か**なければならない**（must）。
- レスポンスメッセージはリソース自体と**すべきである**（should）。ただし、レスポンスで提供する有用なコンテキストがある場合は例外で、その場合レスポンスメッセージはRPC名と一致し、`Response`接尾辞が付か**なければならない**（must）。
  - レスポンスがリソース自体の場合、完全に設定されたリソースを含む**べきである**（should）。
- HTTP動詞は、[カスタムメソッド][aip-136]の通常通り、`POST`と**しなければならない**（must）。
- HTTP URIは`:add*`または`:remove*`で終わら**なければならない**（must）。ここで`*`は、追加または削除されるフィールドのスネークケース単数形である。
- リソース名を受け取るリクエストメッセージフィールドは、URIパスにマッピングする**べきである**（should）。
  - HTTP変数は、`name`や`parent`ではなく、リソースの名前（`book`など）と**すべきである**（should）。
  - その変数はURIパス内の唯一の変数と**すべきである**（should）。
- `google.api.http`アノテーション内のbody句は`"*"`と**すべきである**（should）。
- `Add` RPCで追加されるデータが既に存在する場合、メソッドは`ALREADY_EXISTS`でエラーと**しなければならない**（must）。
- `Remove` RPCで削除されるデータが存在しない場合、メソッドは`NOT_FOUND`でエラーと**しなければならない**（must）。

#### リクエストメッセージ

```proto
message AddAuthorRequest {
  // The name of the book to add an author to.
  string book = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];

  string author = 2 [(google.api.field_behavior) = REQUIRED];
}

message RemoveAuthorRequest {
  // The name of the book to remove an author from.
  string book = 1 [
    (google.api.field_behavior) = REQUIRED,
    (google.api.resource_reference).type = "library.googleapis.com/Book"
  ];

  string author = 2 [(google.api.field_behavior) = REQUIRED];
}
```

- リソースフィールドを含め**なければならない**（must）。これは`name`や`parent`ではなく、リソースの名前（`book`など）と**すべきである**（should）。
  - フィールドは[requiredとしてアノテーションされる][aip-203]**べきである**（should）。
  - フィールドは参照する[リソースタイプ][aip-123]を識別する**べきである**（should）。
- 追加または削除される値のためのフィールドを含め**なければならない**（must）。これはフィールドの単数形と**すべきである**（should）。
  - フィールドは[requiredとしてアノテーションされる][aip-203]**べきである**（should）。
- リクエストメッセージは他のrequiredフィールドを含んでは**ならず**（must not）、またこのAIPまたは別のAIPで説明されているもの以外のoptionalフィールドを含む**べきではない**（should not）。
