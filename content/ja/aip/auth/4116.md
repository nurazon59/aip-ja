# アイデンティティトークン

アイデンティティトークン（IDトークン）は、クライアントまたはエンドユーザーのIDに関する追加クレーム（"email"など）を含むJSON Webトークン（JWT）である。IDトークンに含まれる追加クレームフィールドは、IDトークンプロバイダに依存する。

IDトークンの**取得**は以下のために使用される：

 * `Authentication`ヘッダーに有効なIDトークンを必要とする[保護されたCloud Runサービス][0]へのリクエスト。
 * `Authentication`ヘッダーに有効なIDトークンを必要とする[IAP保護リソース][2]へのリクエスト。

IDトークンの**検証**は以下によって行われるべきである：

 * Google Cloud外から送信された[認証されたPubSubプッシュメッセージ][1]を受信するサービス。

**注:** このAIPは言語に依存しない形でガイダンスと要件を説明するため、特定の言語や環境では不正確または不適切となる可能性がある一般的な用語を使用している。

## ガイダンス

本セクションでは、IDトークンを認証手段として使用するための一般的なガイダンスについて説明する。

### Application Default Credentialsによって取得されるIDトークン

開発者によってtarget audienceが提供された場合、[ADC][3]は提供されたクレデンシャルを使用してIDトークンを取得する**べきである**（should）。例えば、サービスアカウントキーが提供された場合、ADCはOAuthトークンエンドポイントにIDトークンを要求する。[仮想マシンクレデンシャル][4]が提供された場合、ADCは仮想マシンサーバーにIDトークンを要求する。

### IDトークン取得の期待される動作

IDトークンの取得をサポートするために、認証ライブラリは以下の手順に従わ**なければならない**（must）：

1.  Application Default Credentialsが要求されたときに、target audienceパラメータを受け付ける。target audienceとscopeの両方が認証ライブラリに提供された場合、例外をスロー**しなければならない**（must）。

2.  仮想マシンクレデンシャルは、メタデータサーバーの`instance/service-accounts/default/identity`エンドポイントから取得したIDトークンを返し、`audience`クエリストリングパラメータを提供されたtarget audienceに設定する。

3.  サービスアカウントキーは[JWT Bearer][5]グラントタイプを使用し、`target_audience`クレームを提供されたtarget audienceに設定する。これにより、`id_token`キーを含むJSONオブジェクトが返される。

ユーザーアイデンティティフローはIDトークンを発行するために使用されて**もよい**（may）が、必須ではない。

### IDトークン検証の期待される動作

IDトークンの検証をサポートするために、認証ライブラリは以下を実行できるJWTトークンを検証するメソッドを提供**しなければならない**（must）：

1.  ES256署名でJWTを検証する。
2.  ユーザー提供のaudienceに対してJWTの`aud`クレームを検証する。
3.  現在のタイムスタンプに対してJWTの`exp`クレームを検証する。
