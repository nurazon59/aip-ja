# 自己署名JWT

Google API向けの自己署名JWT（JSON Web Token）は2014年に導入され、クライアントアサーションをOAuthトークンと交換する中間ステップをバイパスすることで、サービスアカウント経由でアクセスされるCloud APIのより効率的な認証スタックを提供する。自己署名JWTは、サービスアカウントの秘密鍵によってローカルで署名された、仕様に準拠したJWTである。多くのCloud APIが従来のアクセストークンの代わりに自己署名JWTを受け入れるようになったため、現在では標準的な認証方法と見なされている。

**注:** このAIPは言語に依存しない形でガイダンスと要件を説明するため、特定の言語や環境では不正確または不適切となる可能性がある一般的な用語を使用している。

## ガイダンス

本セクションでは、自己署名JWTを認証手段としてサポートするための一般的なガイダンスについて説明する。

### アプリケーション デフォルト クレデンシャル

自己署名JWTのサポートは、[アプリケーション デフォルト クレデンシャル][0]の一部と見なされる。全体的なフローを理解するには、[AIP-4110][0]を参照のこと。

### Scope vs Audience

従来、自己署名JWTは`audience`でのみ使用可能であった。最近では、Google Cloud認証エコシステムにおいて自己署名JWTでの"scope"クレームの使用が許可され、自己署名JWTがaudienceとscopeの両方で使用可能になった。ただし、潜在的な競合を防ぐため、呼び出し元はscopeとaudienceを同時に指定しては**ならない**（must not）。

#### オプトイン戦略

現在、JWTのこのscopeサポートは、設定可能なブーリアンオプション`UseJWTAccessWithScope`（実際の名前はプログラミング言語によって異なる場合がある）の背後にゲートされている。この機能はすべてのCloud APIサービスのバックエンドでサポートされているわけではないためである。このブーリアンのデフォルト値はfalseであり、自己署名JWTでscopeを使用するには、呼び出し元はこのオプションを`true`に指定**しなければならない**（must）。そうでない場合、scopeが提供されたときは代わりにOAuth交換フローを使用する**べきである**（should）。APIクライアントは、APIに基づいて適切な値でこのオプションを自動的に決定しようと試みる**べきである**（should）ため、この複雑さが上位レイヤーの開発者に露出されることはない。

### サービスアカウントキーのデフォルト認証方法

OAuthフローと比較した優れた効率性と信頼性（交換ステップのバイパス）を考慮して、[ADC][0]はサービスアカウントキーがソースクレデンシャルとして提供された場合、自己署名JWTをデフォルトの認証フローとして使用する**べきである**（should）。

言い換えれば、[ADC][0]は、scopeが提供され、`UseJWTAccessWithScope`を介してJWTが有効になっていない場合にのみ、自己署名JWTフローよりもOAuthフローを選択する**べきである**（should）。

### 期待される動作

自己署名JWTをサポートするために、認証ライブラリは以下の手順に従わ**なければならない**（must）：

1\. [サービスアカウントID JSONファイル][2]を読み込む。自己署名JWTはサービスアカウントIDクレデンシャルタイプのみをサポートすることに注意。

2\. [jwt.io][1]などにある標準的なJWTライブラリを使用して、以下の例のようなヘッダーとペイロードを持つJWTを作成する：

(1) audienceを使用する例：

```json
{
  "alg": "RS256",
  "typ": "JWT",
  "kid": "abcdef1234567890"
}
{
  "iss": "123456-compute@developer.gserviceaccount.com",
  "sub": "123456-compute@developer.gserviceaccount.com",
  "aud": "https://pubsub.googleapis.com/",
  "iat": 1511900000,
  "exp": 1511903600
}
```

(2) scopeを使用する例：

```json
{
  "alg": "RS256",
  "typ": "JWT",
  "kid": "abcdef1234567890"
}
{
  "iss": "123456-compute@developer.gserviceaccount.com",
  "sub": "123456-compute@developer.gserviceaccount.com",
  "scope": "https://www.googleapis.com/auth/cloud-platform",
  "iat": 1511900000,
  "exp": 1511903600
}
```

* ヘッダーの`kid`フィールドには、サービスアカウントの秘密鍵IDを指定する。この値はサービスアカウントJSONファイルの`private_key_id`フィールドに記載されている。

* `iss`および`sub`フィールドには、サービスアカウントのメールアドレスを指定する。この値はサービスアカウントJSONファイルの`client_email`フィールドに記載されている。

* `aud`フィールドには、audienceを指定する。デフォルトのaudience値は`https://[SERVICE]/`である**べきである**（should）（例：`https://pubsub.googleapis.com/`）。

* `scope`フィールドには、スコープを指定する。`aud`と`scope`は同時に指定できないことに注意。

* `iat`フィールドには現在のUnix時刻を指定し、`exp`フィールドの値はちょうど3600秒後（JWTの有効期限）で**なければならない**（must）。

3\. サービスアカウントJSONファイルの秘密鍵を使用して、JWTをRSA-256で署名する。
