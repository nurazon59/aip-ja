# 外部アカウントクレデンシャル（ワークロード アイデンティティ フェデレーション）

ワークロード アイデンティティ フェデレーションを使用すると、アプリケーションはAmazon Web Services（AWS）、Microsoft Azure、またはOpenID Connect（OIDC）やSAML 2.0をサポートする任意のアイデンティティプロバイダからGoogle Cloudリソースにアクセスできる。

従来、Google Cloud外で実行されるアプリケーションは、サービスアカウントキーを使用してGoogle Cloudリソースにアクセスしていた。アイデンティティ フェデレーションを使用すると、ワークロードにサービスアカウントを借用させることができる。これにより、Google Cloudリソースに直接アクセスできるようになり、サービスアカウントキーに関連するメンテナンスとセキュリティの負担がなくなる。

**注:** このAIPは言語に依存しない形でガイダンスと要件を説明するため、特定の言語や環境では不正確または不適切となる可能性がある一般的な用語を使用している。

## ガイダンス

本セクションでは、非Google外部クレデンシャル（AWS、Azure、OIDCおよびSAML IdPなど）を認証手段としてサポートするための一般的なガイダンスについて説明する。

### 前提条件

非Googleクラウドプラットフォームからワークロード アイデンティティ フェデレーションを使用してGoogle Cloudリソースにアクセスするには、ワークロード アイデンティティプール、プロバイダ、サービスアカウントの借用を構成し、認証ライブラリが使用するJSON設定ファイルを生成するために、以下の手順が必要である。

- [AWSからのワークロード アイデンティティ フェデレーションの構成][0]
- [Microsoft Azureからのワークロード アイデンティティ フェデレーションの構成][1]
- [OIDCアイデンティティプロバイダからのワークロード アイデンティティ フェデレーションの構成][2]
- [SAMLアイデンティティプロバイダからのワークロード アイデンティティ フェデレーションの構成][3]

### 設定ファイルの生成と使用

ワークロード アイデンティティ フェデレーションが構成された後、JSON設定ファイルを生成する**べきである**（should）。以下のサンプルは、AWS設定ファイルがどのように生成されるかを示している：

```bash
$ gcloud iam workload-identity-pools create-cred-config \
    projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID \
    --service-account=$SERVICE_ACCOUNT_EMAIL \
    --aws \
    --output-file=$FILEPATH.json
```

以下の値を置き換える必要がある：

- **PROJECT_NUMBER**: ワークロード アイデンティティプールを含むプロジェクトのプロジェクト番号。
- **POOL_ID**: ワークロード アイデンティティプールのID。
- **PROVIDER_ID**: ワークロード アイデンティティプールプロバイダのID。
- **SERVICE_ACCOUNT_EMAIL**: 借用するサービスアカウントのメールアドレス。
- **FILEPATH**: 設定を保存するファイルパス。

[AWS IMDSv2](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/configuring-instance-metadata-service.html) を使用している場合、[`gcloud iam workload-identity-pools create-cred-config`](/sdk/gcloud/reference/iam/workload-identity-pools/create-cred-config) コマンドに追加フラグ `--enable-imdsv2` を追加する**べきである**（should）：

```bash
$ gcloud iam workload-identity-pools create-cred-config \
    projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID \
    --service-account=$SERVICE_ACCOUNT_EMAIL \
    --aws \
    --enable-imdsv2 \
    --output-file=$FILEPATH.json
```

サービスアカウントのアクセストークンの有効期間を構成したい場合、[`gcloud iam workload-identity-pools create-cred-config`](/sdk/gcloud/reference/iam/workload-identity-pools/create-cred-config) コマンドに追加フラグ `--service-account-token-lifetime-seconds` を追加する**べきである**（should）（この例ではAWS設定を使用しているが、トークンの有効期間はすべてのワークロード アイデンティティ フェデレーションプロバイダで構成可能）：

```bash
$ gcloud iam workload-identity-pools create-cred-config \
    projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID \
    --service-account=$SERVICE_ACCOUNT_EMAIL \
    --aws \
    --service-account-token-lifetime-seconds=$TOKEN_LIFETIME \
    --output-file=$FILEPATH.json
```

service-account-token-lifetime-secondsフラグはoptionalである。指定しない場合、デフォルトは1時間である。最小許容値は600（10分）、最大許容値は43200（12時間）である。1時間を超える有効期間が必要な場合、サービスアカウントは `constraints/iam.allowServiceAccountCredentialLifetimeExtension` 制約を実施する組織ポリシーで許可された値として追加されなければならない。

外部アイデンティティ設定ファイルは[Application Default Credentials][6]で使用できる。外部アイデンティティをApplication Default Credentialsで使用するには、このファイルのフルパスを `GOOGLE_APPLICATION_CREDENTIALS` 環境変数に格納する**べきである**（should）。

```bash
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/config.json
```

これで、ライブラリは設定ファイルで提供されたコンテキストから自動的に適切なタイプのクライアントを選択し、クレデンシャルを初期化できる：

```python
import google.auth

credentials, project = google.auth.default()
```

外部アカウントクレデンシャルは、生成された設定ファイルを使用して明示的に初期化することもできる。

```python
# AzureまたはOIDC/SAMLプロバイダ向けのサンプル。
import json

from google.auth import identity_pool

json_config_info = json.loads(function_to_get_json_config())
credentials = identity_pool.Credentials.from_info(json_config_info)
scoped_credentials = credentials.with_scopes(
    ['https://www.googleapis.com/auth/cloud-platform'])
```

```python
# AWS向けのサンプル。
import json

from google.auth import aws

json_config_info = json.loads(function_to_get_json_config())
credentials = aws.Credentials.from_info(json_config_info)
scoped_credentials = credentials.with_scopes(
    ['https://www.googleapis.com/auth/cloud-platform'])
```

## 期待される動作

認証ライブラリはJSON設定ファイルの情報を使用して、外部クレデンシャルを取得し、GCP Security Token Service（トークン交換エンドポイント `https://sts.googleapis.com/v1/token` 経由）を使用してGoogleアクセストークンと交換し、その後 **IamCredentials** の [generateAccessToken][5] APIを呼び出してサービスアカウントを借用することでGCPリソースにアクセスする**べきである**（should）。

すべての外部アカウントJSONファイルは以下のフィールドを共有**しなければならない**（must）：

| フィールド名                       | Required | 説明 |
|-----------------------------------|----------|:------------|
| type                              | Yes      | 新しいタイプのクレデンシャルオブジェクトを識別する。これは "external_account" でなければならない。 |
| audience                          | Yes      | ワークロード アイデンティティプールとそのプール内のプロバイダ識別子のリソース名を含むSTS audience。 |
| subject_token_type                | Yes      | [OAuth 2.0トークン交換仕様][7]に基づくSTS subject token type。 |
| service_account_impersonation_url | No       | サービスアカウント借用リクエストのURL。これが利用できない場合、STSから返されたアクセストークンを借用なしで直接使用する**べきである**（should）。 |
| service_account_impersonation.*   | No       | 追加のサービスアカウント借用オプションを定義するオブジェクト。現在サポートされているフィールドは1つのみ："token_lifetime_seconds"：要求されたアクセストークンの有効期間（例：`2800`）。 |
| token_url                         | Yes      | STSトークン交換エンドポイント。 |
| credential_source.*               | Yes      | ローカル環境から外部クレデンシャルを取得するためのメカニズムを定義するオブジェクト。STSエンドポイント経由でGCPアクセストークンと交換するために使用される。 |

認証ライブラリとアプリケーションは、すべてのタイプの外部アカウントクレデンシャルについて以下の手順に従わ**なければならない**（must）：

- **credential_source** を確認して、STSエンドポイントに渡すsubject tokenを構築するために使用する外部クレデンシャルを取得するために必要なロジックを決定する。これは以下ですべてのクレデンシャル設定について詳細に説明される。
- [rfc8693][4]に基づいてSTSリクエストを構築する：
  - STS audienceは **audience** フィールドを使用して構築する**べきである**（should）。
  - **grant_type** は `urn:ietf:params:oauth:grant-type:token-exchange` で**なければならない**（must）。
  - **requested_token_type** は `urn:ietf:params:oauth:token-type:access_token` で**なければならない**（must）。
  - **subject_token_type** はRFCで説明されている **subject_token_type** フィールドである。
  - **subject_token** は取得された外部クレデンシャルである。これが様々な環境でどのように取得されるかについては後続のセクションを参照のこと。
  - **scope**：トークンが使用されるべきサービスまたはリソースのコンテキストにおいて、要求されたセキュリティトークンの望ましいスコープを指定する、スペース区切りで大文字小文字を区別しないOAuthスコープのリスト。サービスアカウントの借用が使用される場合、クラウドプラットフォームまたはIAMスコープをSTSに渡し、顧客提供のスコープは **IamCredentials** の [generateAccessToken][5] 呼び出しで渡す**べきである**（should）。
  - STSトークン交換URLは **token_url**（例：`https://sts.googleapis.com/v1/token`）である**べきである**（should）。
- STSトークン交換リクエストを送信してGoogleアクセストークンとその有効期限を取得する。
- **service_account_impersonation_url** が利用可能な場合、そのエンドポイントに以前返されたGoogleアクセストークンをPOSTしてサービスアカウント借用フローを開始する。
  - これが利用できない場合、フローを終了し、認可にSTSアクセストークンをそのまま使用する。
  - このエンドポイントにはスコープのリストも提供する必要がある。このエンドポイントには顧客提供のスコープを使用する**べきである**（should）。
  - このAPIにアクセスするには、基盤となるアクセストークンに（Cloudプラットフォーム `https://www.googleapis.com/auth/cloud-platform` またはIAMスコープ `https://www.googleapis.com/auth/iam`）が必要である。
  - このエンドポイントにはサービスアカウントのアクセストークンの有効期間も指定する必要がある。**service_account_impersonation.token_lifetime_seconds** の値が提供されていればそれが使用され、そうでなければデフォルトの1時間となる。

### AWSにおけるsubject tokenの決定

外部アカウント設定JSONファイルは、GCP STSトークン交換エンドポイントにsubject tokenとして渡すAWSクレデンシャルの取得を容易にするために、`credential_source` オブジェクトに以下の情報を含む**べきである**（should）。

| フィールド名                                       | Required | 説明 |
|--------------------------------------------------|----------|:------------|
| environment_id                 | Yes      | `aws${version}` 形式の環境識別子。基盤となるAWS実装に破壊的変更が導入されたかどうかを認証ライブラリに示すためにバージョンを指定する**べきである**（should）。そのため、ライブラリの現在のバージョンでaws1がサポートされているが、aws2のクレデンシャルファイルが提供された場合、開発者にライブラリの新しいバージョンへのアップグレードを指示するエラーをスローする**べきである**（should）。 |
| region_url                     | No       | このURLは、リージョン環境変数が存在しない場合に、署名付きリクエストの構築に必要な現在のAWSリージョンを決定するために使用される**べきである**（should）。 |
| url                            | No       | このAWSメタデータサーバーURLは、`GetCallerIdentity` リクエストの署名に必要なアクセスキー、シークレットキー、セキュリティトークンを取得するために使用される**べきである**（should）。$ROLE_NAME は、パラメータなしでこのエンドポイントを呼び出し、返されたロール名をこのURLに追加して再度呼び出すことで取得する：http://169.254.169.254/latest/meta-data/iam/security-credentials/$ROLE_NAME |
| regional_cred_verification_url | Yes      | リージョナルAWS `GetCallerIdentity` アクションURLを定義する。このURLはAWSアカウントIDとそのロールを決定するために使用される**べきである**（should）。これは実際には認証ライブラリによって呼び出されるべきではない。STSトークンサーバー上で呼び出される**べきである**（should）。リージョンはSDKによって置換される**べきである**（should）（例：`sts.eu-west-1.amazonaws.com`）。 |
| imdsv2_session_token_url       | No       | このURLの存在は、認証ライブラリにAWSからセッショントークンを取得することを強制する。このフィールドはIMDSv2を使用するEC2インスタンスに必要である。このセッショントークンは、メタデータエンドポイントへの呼び出し時に使用される。 |

AWS設定ファイルのJSONは以下の形式を持つ**べきである**（should）：

```json
{
  "type": "external_account",
  "audience": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
  "subject_token_type": "urn:ietf:params:aws:token-type:aws4_request",
  "service_account_impersonation_url": "https://iamcredentials.googleapis.com/v1/projects/-/serviceAccounts/$EMAIL:generateAccessToken",
  "token_url": "https://sts.googleapis.com/v1/token",
  "credential_source": {
    "environment_id": "aws1",
    "region_url": "http://169.254.169.254/latest/meta-data/placement/availability-zone",
    "url": "http://169.254.169.254/latest/meta-data/iam/security-credentials",
    "regional_cred_verification_url": "https://sts.{region}.amazonaws.com?Action=GetCallerIdentity&Version=2011-06-15",
    "imdsv2_session_token_url": "http://169.254.169.254/latest/api/token"
  }
}
```

認証ライブラリとアプリケーションは以下の手順に従わ**なければならない**（must）：

- **credential_source** のenvironment IDを確認する。environment IDが `aws${version}` の場合、これはAWSネイティブクレデンシャルである**べきである**（should）。
- environment IDのバージョンを検査する。これが新しい予期しないエラーの場合、認証ライブラリをこのタイプのクレデンシャルを処理するように更新する必要があるというエラーを発生させる。
- **url**、**regional_url**、**imdsv2_session_token_url** フィールドのホストが提供されている場合、それらを検証する。ホストは **169.254.169.254** または **fd00:ec2::254** のいずれかである**べきである**（should）。
- **imdsv2_session_token_url** が利用可能な場合、**imdsv2_session_token_url** からセッショントークンを取得する。注：このステップは、リージョンやセキュリティクレデンシャルを取得するためにメタデータサーバーと通信する必要がある場合にのみ実行する。
- 環境変数を以下の順序で確認して（`AWS_REGION`、次に `AWS_DEFAULT_REGION`）AWSリージョンを決定する。見つかった場合、AWSメタデータサーバーを使用してこの値を決定するのをスキップする。
- リージョン環境変数が提供されていない場合、**region_url** を使用して現在のAWSリージョンを決定する。APIはゾーン名を返す（例：`us-east-1d`）。リージョンは最後の文字を削除して決定する**べきである**（should）（例：`us-east-1`）。
- 環境変数 `AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY` および optional の `AWS_SESSION_TOKEN` をAWSセキュリティクレデンシャルとして確認する。見つかった場合、AWSメタデータサーバーを使用してこれらの値を決定するのをスキップする。
- **url** が利用可能で、セキュリティクレデンシャルの環境変数が提供されていない場合：
  - **url** を呼び出して、現在のインスタンスにアタッチされたAWS IAMロール名を取得する。
  - **url/$ROLE_NAME** を呼び出して、`GetCallerIdentity` リクエストの署名に必要なアクセスキー、シークレットキー、セキュリティトークンを取得する。
- [GetCallerIdentity][9] **regional_cred_verification_url**（リージョンが置換されたもの）を使用してAWS署名付きリクエスト（[AWS Signature Version 4][8]）を構築する。これはURLエンコードされたJSONとしてフォーマットしてシリアライズし、**subject_token** としてSTSエンドポイントに渡す**べきである**（should）。以下は使用されるJSON形式のサンプルである：

  ```json
  {
    "url": "https://sts.us-east-1.amazonaws.com?Action=GetCallerIdentity&Version=2011-06-15",
    "headers": [
      {
        "value": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
        "key": "x-goog-cloud-target-resource"
      },
      {
        "value": "20200228T225005Z",
        "key": "x-amz-date"
      },
      {
        "value": "AWS4-HMAC-SHA256 Credential=AKIASOZTBDV4D7ABCDEDF/20200228/us-east-1/sts/aws4_request, SignedHeaders=host;x-amz-date;x-amz-security-token;x-goog-cloud-target-resource, Signature=abcedefdfedfd",
        "key": "Authorization"
      },
      {
        "value": "sts.us-east-1.amazonaws.com",
        "key": "host"
      },
      {
        "value": "IQoJb3JpZ2luX2VjEIz//////////wEaCXVzLWVh...",
        "key": "x-amz-security-token"
      }
    ],
    "method": "POST",
    "body": ""
  }
  ```
  AWSトークンの場合、STSは特定のワークロード アイデンティティプロバイダ用のトークンであることを認識するために、特別なヘッダー `x-goog-cloud-target-resource` を必要とする。

### Microsoft AzureおよびURLソースクレデンシャルにおけるsubject tokenの決定

外部アカウント設定JSONファイルは、Azureおよび他のURLソースクレデンシャルをGCP STSトークン交換エンドポイントにsubject tokenとして渡すための取得を容易にするために、`credential_source` オブジェクトに以下の情報を含む**べきである**（should）。

| フィールド名                                        | Required | 説明 |
|---------------------------------------------------|----------|:-------------|
| url                             | Yes      | 外部クレデンシャルを取得するためのローカルメタデータサーバーを定義する。Azureの場合、これはAzure ADアクセストークンを取得するために使用されるAzure Instance Metadata Service（IMDS）URLである**べきである**（should）。 |
| headers                         | No       | credential_source.url へのGETリクエストに追加するヘッダーを定義する。 |
| format.type                     | No       | URL応答の形式を示す。"text" または "json" のいずれかである。デフォルトは "text" である**べきである**（should）。 |
| format.subject_token_field_name | No       | JSON URL応答に必要。subject_token が格納されるJSONフィールド名を示す。 |

URLソース設定ファイル（OIDC / SAML）のJSONは以下の形式を持つ**べきである**（should）：

```json
{
  "type": "external_account",
  "audience": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
  "subject_token_type": "urn:ietf:params:oauth:token-type:jwt",
  "service_account_impersonation_url": "https://iamcredentials.googleapis.com/v1/projects/-/serviceAccounts/$EMAIL:generateAccessToken",
  "token_url": "https://sts.googleapis.com/v1/token",
  "credential_source": {
    "url": "http://localhost:5000/token",
    "format": {
      "type": "json",
      "subject_token_field_name": "id_token"
    }
  }
}
```

Azure設定ファイルはOIDC URLソースクレデンシャルの一種である。

```json
{
  "type": "external_account",
  "audience": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
  "subject_token_type": "urn:ietf:params:oauth:token-type:jwt",
  "service_account_impersonation_url": "https://iamcredentials.googleapis.com/v1/projects/-/serviceAccounts/$EMAIL:generateAccessToken",
  "token_url": "https://sts.googleapis.com/v1/token",
  "credential_source": {
    "headers": {
      "Metadata": "True"
    },
    "url": "http://169.254.169.254/metadata/identity/oauth2/token?api-version=2018-02-01&resource=https://iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
    "format": {
      "type": "json",
      "subject_token_field_name": "access_token"
    }
  }
}
```

認証ライブラリとアプリケーションは以下の手順に従わ**なければならない**（must）：

- **credential_source** に **url** フィールドがあり、**environment_id** がないことを確認する。それ以外の場合は残りのステップをスキップする。
- このローカル **url** にHTTP GETリクエストを送信し、設定ファイルで提供されている場合、**headers** のキー/値をリクエストヘッダーに注入する。リクエストは、STSトークンエンドポイントに渡す外部クレデンシャルのsubject tokenで応答する**べきである**（should）。
- トークンを解析する前に、**format** フィールドを確認する。
- **format** が利用できない場合、URL応答によって返された外部クレデンシャルはプレーンテキスト形式で提供されると想定する。
- 利用可能な場合、typeが **json** かどうかを確認する。
  - jsonの場合、**subject_token_field_name** を確認する。Azureの場合、これは **access_token** に設定される。
  - ファイルをJSONとして解析し、**subject_token_field_name** の値に基づいてフィールド名から外部クレデンシャルを取得する。

### ファイルソースクレデンシャルにおけるsubject tokenの決定

外部アカウント設定JSONファイルは、GCP STSトークン交換エンドポイントにsubject tokenとして渡すファイルソースクレデンシャルの取得を容易にするために、`credential_source` オブジェクトに以下の情報を含む。

| フィールド名                                        | Required | 説明 |
|---------------------------------------------------|----------|:------------|
| file                            | Yes      | クレデンシャルのソース。ローカルで利用可能なクレデンシャルに使用される**べきである**（should）。両方が提供された場合、`url` より優先される**べきである**（should）。 |
| format.type                     | No       | トークンが保存されているファイルの形式を示す。"text" または "json" のいずれかである。デフォルトは "text" である**べきである**（should）。 |
| format.subject_token_field_name | No       | JSONファイル形式の場合に必要。`subject_token` が格納されるJSONフィールド名を示す。 |

ファイルソース設定ファイル（OIDC / SAML）のJSONは以下の形式を持つ**べきである**（should）：

```json
{
  "type": "external_account",
  "audience": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
  "subject_token_type": "urn:ietf:params:oauth:token-type:saml2",
  "service_account_impersonation_url": "https://iamcredentials.googleapis.com/v1/projects/-/serviceAccounts/$EMAIL:generateAccessToken",
  "token_url": "https://sts.googleapis.com/v1/token",
  "credential_source": {
    "file": "/var/run/saml/assertion/token"
  }
}
```

認証ライブラリとアプリケーションは以下の手順に従わ**なければならない**（must）：

- **credential_source** に **file** フィールドがあり、**environment_id** がないことを確認する。そうでない場合、これはファイルソースクレデンシャルではなく、以降の手順は適用されない。
- `credential_source.file` フィールドで指定されたファイルの場所から外部クレデンシャルを取得する。
- トークンを解析する前に、**format** フィールドを確認する。
- **format** が利用できない場合、外部クレデンシャルはプレーンテキスト形式で提供されると想定する。
- 利用可能な場合、typeが **json** かどうかを確認する。
  - jsonの場合、**subject_token_field_name** を確認する。
  - ファイルをJSONとして解析し、**subject_token_field_name** の値に基づいてフィールド名から外部クレデンシャルを取得する。


### 実行可能ソースクレデンシャルにおけるsubject tokenの決定

外部アカウント設定JSONファイルは、GCP STSトークン交換エンドポイントにsubject tokenとして渡す実行可能ソースクレデンシャルの取得を容易にするために、`credential_source` オブジェクトに以下の情報を含む。

| フィールド名                | Required | 説明                                                                                                                                                                                                                                                                                              |
|---------------------------|----------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| executable                | Yes      | 実行可能ファイルを実行するために必要な情報を保持する。                                                                                                                                                                                                                                                   |
| executable.command        | Yes      | subject tokenを取得するために実行する完全なコマンドを指定する。引数を含めることができる。プログラムの絶対パスでなければならない。                                                                                                                                                                   |
| executable.timeout_millis | No       | タイムアウト期間をミリ秒単位で指定する。指定されない場合のデフォルトは30秒。                                                                                                                                                                                                               |
| executable.output_file    | No       | 実行可能ファイルが応答をキャッシュする出力ファイルの絶対パスを指定する。このパスを指定することで、認証ライブラリは実行可能ファイルを実行する前にまずこの場所を確認する。ファイルの形式は、以下で定義する認証ライブラリが期待するJSON形式と一致する**べきである**（should）。 |

実行可能ソース設定ファイル（OIDC / SAML）のJSONは以下の形式を持つ**べきである**（should）：

```json
{
  "type": "external_account",
  "audience": "//iam.googleapis.com/projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/$POOL_ID/providers/$PROVIDER_ID",
  "subject_token_type": "urn:ietf:params:oauth:token-type:saml2",
  "token_url": "https://sts.googleapis.com/v1/token",
  "service_account_impersonation_url": "https://iamcredentials.googleapis.com/v1/projects/-/serviceAccounts/$EMAIL@project.iam.gserviceaccount.com:generateAccessToken",
  "credential_source": {
    "executable": {
      "command": "/path/to/executable --arg1=value1 --arg2=value2",
      "timeout_millis": 5000,
      "output_file": "/path/to/cached/credentials"
    }
  }
}
```

実行可能ソースクレデンシャルを使用するには、`GOOGLE_EXTERNAL_ACCOUNT_ALLOW_EXECUTABLES` 環境変数を `1` に設定**しなければならない**（must）。

さらに、実行可能ファイルは以下の応答形式に従わ**なければならない**（must）：

成功応答：

| フィールド名                | タイプ    | 説明                                                                                                                                                                        |
|---------------------------|---------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| version                   | number  | JSON出力のバージョン。現在はバージョン1のみサポートされている。                                                                                                             |
| success                   | boolean | 応答のステータス。この場合はtrue。                                                                                                                                     |
| token_type                | string  | サードパーティのsubject token type。*urn:ietf:params:oauth:token-type:jwt*、*urn:ietf:params:oauth:token-type:id_token*、または *urn:ietf:params:oauth:token-type:saml2* でなければならない。 |
| id_token OR saml_response | string  | サードパーティのOIDCトークンまたはSAML応答。                                                                                                                                         |
| expiration_time           | number  | オプションのサードパーティsubject tokenの有効期限（Unixエポック時間、秒単位）。クレデンシャル設定で出力ファイルが指定されている場合にのみ応答で必要。 |

成功した実行可能OIDC応答のサンプル：
```json
{
  "version": 1,
  "success": true,
  "token_type": "urn:ietf:params:oauth:token-type:id_token",
  "id_token": "...",
  "expiration_time": 1620499962
}
```

成功した実行可能SAML応答のサンプル：
```json
{
  "version": 1,
  "success": true,
  "token_type": "urn:ietf:params:oauth:token-type:saml2",
  "saml_response": "...",
  "expiration_time": 1620499962
}
```

エラー応答：

| フィールド名 | タイプ    | 説明                                                            |
|------------|---------|:-----------------------------------------------------------------------|
| version    | number  | JSON出力のバージョン。現在はバージョン1のみサポートされている。 |
| success    | boolean | 応答のステータス。この場合はfalse。                        |
| code       | string  | エラーコード。                                                        |
| message    | string  | エラーメッセージ。                                                     |

実行可能エラー応答のサンプル：
```json
{
  "version": 1,
  "success": false,
  "code": "401",
  "message": "Caller not authorized."
}
```

認証ライブラリとアプリケーションは以下の手順に従わ**なければならない**（must）：

- **credential_source** に **executable** フィールドがあり、**environment_id** がないことを確認する。そうでない場合、これは実行可能ソースクレデンシャルではなく、以降の手順は適用されない。
- **credential_source.executable** フィールドから外部クレデンシャルの実行可能情報を取得する。
- `GOOGLE_EXTERNAL_ACCOUNT_ALLOW_EXECUTABLES` 環境変数が **1** に設定されていることを確認する。設定されていない場合、エラーを返す。
- 次のステップの前に、**credential_source.executable.output_file** がクレデンシャル設定で指定されているかを確認する。
  - 指定されている場合、その場所に実行可能応答があるかを確認する。
  - 応答が有効で有効期限が切れていない場合、またはその場所に応答がない場合、実行を続行する。
  - 応答が不正形式または無効の場合、エラーを返す。
- 以下の環境変数が実行可能ファイルから利用可能であることを確認する：
  - `GOOGLE_EXTERNAL_ACCOUNT_AUDIENCE`: クレデンシャル設定からのaudienceフィールド。常に存在しなければならない。
  - `GOOGLE_EXTERNAL_ACCOUNT_TOKEN_TYPE`: subject token type。常に存在しなければならない。
  - `GOOGLE_EXTERNAL_ACCOUNT_IMPERSONATED_EMAIL`: サービスアカウントのメールアドレス。サービスアカウントの借用が使用される場合のみ存在する。
  - `GOOGLE_EXTERNAL_ACCOUNT_OUTPUT_FILE`: クレデンシャル設定からの出力ファイルの場所。クレデンシャル設定で指定されている場合のみ存在する。
- **credential_source.executable.command** で指定されたコマンドを実行する。
  - 以下のシナリオで失敗とする：
    - 実行可能ファイルが指定されたタイムアウト期間内に完了しなかった。
    - 実行可能ファイルの応答が無効、失敗、または期限切れであった。
    - 実行可能ファイルがゼロ以外の終了コードで終了した。
- 実行可能応答をJSONとして解析し、**token_type** の値に基づいてフィールド名から外部クレデンシャルを取得する。
  - token_typeの値は **urn:ietf:params:oauth:token-type:jwt**、**urn:ietf:params:oauth:token-type:id_token**、または **urn:ietf:params:oauth:token-type:saml2** でなければならない。
  - **token_type** が **urn:ietf:params:oauth:token-type:saml2** の場合、subject tokenは **saml_response** フィールドから解析される。
  - それ以外の場合は **id_token** フィールドから解析される。
