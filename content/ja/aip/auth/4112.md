# サービスアカウントキー

サービスアカウントは、人間ではなくアプリケーションや仮想マシン（VM）インスタンスによって使用される特別な種類のアカウントである。アプリケーションはサービスアカウントを使用して、認可されたAPI呼び出しを行う。この認可は、サービスアカウント自体として、またはドメイン全体の委任を通じてGoogle WorkspaceやCloud Identityユーザーとして行われる。サービスアカウントには、認証に使用される秘密鍵/公開鍵のRSAキーペアが関連付けられており、これがこのAIPの焦点である。

**注:** このAIPは言語に依存しない形でガイダンスと要件を説明するため、特定の言語や環境では不正確または不適切となる可能性がある一般的な用語を使用している。

## ガイダンス

本セクションでは、従来のOAuthフローを使用したサービスアカウントキー認証をサポートするための一般的なガイダンスについて説明する。自己署名JWTフローを使用したサービスアカウントキー認証については、[自己署名JWT][0]のAIPを参照のこと。

### OAuthを使用したサービスアカウントキー認証

最新の[ADC][1]ガイダンスでは、サービスアカウントキーがソースクレデンシャルとして提供された場合、デフォルトではOAuthの"scope"引数を使用しない[自己署名JWT][0]フローを使用することが推奨される。したがって、クライアントはOAuth "scope"がクライアントに明示的に提供された場合にのみ、自己署名JWTフローよりもOAuthフローを選択する**べきである**（should）。

### 期待される動作

OAuthフローを使用したサービスアカウントキー認証をサポートするために、認証ライブラリは以下の手順に従わ**なければならない**（must）：

1\. **GOOGLE_APPLICATION_CREDENTIALS**環境変数、またはクライアントオプションを通じて指定された明示的な**credentials**ファイルパスから、**サービスアカウントID JSONファイル**を読み込む。JSONファイルは以下のようになる：

```json
{
  "type": "service_account",
  "project_id": "testproject",
  "private_key_id": "redacted",
  "private_key": "redacted",
  "client_email": "testserviceaccount@test.iam.gserviceaccount.com",
  "client_id": "113258942105700140798",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
  "client_x509_cert_url": "https://www.googleapis.com/robot/v1/metadata/x509/testserviceaccount%40test.iam.gserviceaccount.com"
}
```

**注:** サービスアカウントID JSONファイルは[Google Cloudコンソール][2]からダウンロードできる。

2\. [jwt.io][3]などにある標準的なJWTライブラリを使用して、以下の例のようなヘッダーとクレームセットを持つJWTを作成する：

```json
{
  "alg": "RS256",
  "typ": "JWT",
  "kid": "abcdef1234567890"
}
{
  "iss": "123456-compute@developer.gserviceaccount.com",
  "sub": "123456-compute@developer.gserviceaccount.com",
  "scope": "https://www.googleapis.com/auth/pubsub",
}
```

* ヘッダーの`kid`フィールドには、サービスアカウントの秘密鍵IDを指定する。この値はサービスアカウントJSONファイルの**private_key_id**フィールドに記載されている。

* `iss`および`sub`フィールドには、サービスアカウントのメールアドレスを指定する。この値はサービスアカウントJSONファイルの**client_email**フィールドに記載されている。

* `scope`フィールドには、クライアントによって提供された**scope**パラメータを指定する。複数のスコープは単一スペース（" "）を区切り文字として指定できる。

3\. サービスアカウントJSONファイルの**private_key**を使用して、JWTをRSA-256で署名する。出力は2-legged OAuthフローのリクエストペイロード（別名アサーション）として使用される。

4\. パラメータ"grant_type"を"urn:ietf:params:oauth:grant-type:jwt-bearer"として定義する。パラメータ"assertion"をステップ3のペイロードとして定義する。

5\. ステップ4のパラメータを使用して**token_uri**エンドポイントにPOSTリクエストを送信し、OAuth2アクセストークンを取得する。
