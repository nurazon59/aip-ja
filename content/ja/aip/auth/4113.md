# gcloud CLI統合

[gcloud][0]は、認証、ローカル設定、開発者ワークフロー、およびGoogle Cloud Platform（GCP）APIとの対話を管理するCLIである。gcloudは、3-legged OAuth（3LO）Webフローを介してユーザーアクセスクレデンシャルを取得し、それらをJSON形式でwell-knownな場所に保存することで、デフォルトクレデンシャルを生成できる。これらのクレデンシャルは、gcloud以外のアプリケーションによって消費されることを意図している。[Application Default Credentials（ADC）およびGoogle Unified Auth Clients（GUAC）][1]は、gcloudのデフォルトクレデンシャルをサポート**しなければならない**（must）。

**注:** サンプルコードはPythonで記述されているが、このAIPは言語に依存しない形でガイダンスと要件を説明する。特定の言語や環境では不正確または不適切となる可能性がある一般的な用語を使用している。

## ガイダンス

本セクションでは、gcloudのデフォルトクレデンシャルをサポートするための一般的なガイダンスについて説明する。

### クレデンシャルの生成

gcloudのデフォルトクレデンシャルは、コマンド'[gcloud auth application-default login][2]'を使用して生成できる。

```
$ gcloud auth application-default login
```

生成されたクレデンシャルは、プラットフォームごとに異なるwell-knownな場所に保存される：

- **Linux、Mac**: \$HOME/.config/gcloud/application_default_credentials.json
- **Windows**: %APPDATA%/gcloud/application_default_credentials.json

以下はgcloudのデフォルトクレデンシャルの例である：

```json
{
  "client_id": "fake_id.apps.googleusercontent.com",
  "client_secret": "fake_secret",
  "quota_project_id": "fake_project",
  "refresh_token": "fake_token",
  "type": "authorized_user"
}
```

すべてのフィールドは、'quota_project_id'を除き、Google認証バックエンドからのログイン応答によって設定される。'quota_project_id'はgcloudのコンテキストから取得される。さらに、ユーザーは'--client-id-file'フラグを使用して'quota_project_id'を上書きできる：

```
$ gcloud auth application-default login --client-id-file=clientid.json
```

### 期待される動作

認証ライブラリはgcloudのデフォルトクレデンシャルの情報を使用して、Google認証バックエンドとアクセストークンを交換する。結果として得られるアクセストークンは、アプリケーションがGCP APIを呼び出すためにさらに使用される。

認証ライブラリとアプリケーションは以下の手順に従わ**なければならない**（must）：

- 認証ライブラリは、実行中のプラットフォームのwell-knownな場所（前の[セクション](#credentials-generation)を参照）からgcloudのデフォルトクレデンシャルを読み込む。

- 認証ライブラリは、gcloudのデフォルトクレデンシャル内のリフレッシュトークン、クライアントID、クライアントシークレットを使用してGoogle認証バックエンドを呼び出し、アクセストークンを取得する。Google認証バックエンドからアクセストークンを取得するためのURLはhttps://oauth2.googleapis.com/tokenである。以下は、認証ライブラリとして[google-auth][3]、HTTPトランスポートとして[urllib3][4]を使用したコード例である。

```python
from google.oauth2 import _client as google_auth_client
import google.auth.transport.urllib3 as google_auth_urllib3
import urllib3

# Google認証バックエンドと通信するためのHTTPトランスポートを作成する。
http = urllib3.PoolManager()
request = google_auth_urllib3.Request(http)

# アクセストークンリクエストのパラメータを構築する。gcloudのデフォルト
# クレデンシャルは前のステップで辞書'gcloud_default'に
# デシリアライズされているものとする。
token_uri = 'https://oauth2.googleapis.com/token'
refresh_token = gcloud_default['refresh_token']
client_id = gcloud_default['client_id']
client_secret = gcloud_default['client_secret']
scopes = ['https://www.googleapis.com/auth/cloud-platform']

# アクセストークンを取得する。
access_token, _, _, _ = google_auth_client.refresh_grant(
	request, token_uri, refresh_token, client_id, client_secret, scopes)
```

- アプリケーションは前のステップで取得したアクセストークンを使用してGCP APIを呼び出す。gcloudのデフォルトクレデンシャル内の'quota_project_id'は、'X-Goog-User-Project' HTTPヘッダーに追加する**べきである**（should）。これにより、関連付けられたアカウントに請求とクォータが課金される。
