# Automatically populate fields in the request message

[AIP-155] で説明されているリクエストの冪等性を活用するAPIでは、APIは `request_id` などのフィールドが顧客によってまだ設定されていない場合に、client libraryに自動的にそれらを設定させることができる。

**Note:** この機能は主にリクエストメッセージ内の `request_id` フィールドのために書かれている。とはいえ、この機能は**しなければならない**（must）以下の条件が満たされる限り、任意の名前のフィールドで動作する。

## Guidance

APIは**してもよい**（may）リクエストメッセージ内のフィールドを自動的に設定するように構成してもよい。フィールドが自動的に設定されるためには、以下の条件が **すべて** 真で**なければならない**（must）：

- フィールドは**しなければならない**（must）`string` 型である
- フィールドは**しなければならない**（must）リクエストメッセージのトップレベルにある
- RPCは**しなければならない**（must）unary RPCである（つまり、ストリーミングRPCはサポートされない）
- フィールドは**してはならない**（must not）[`google.api.field_behavior = REQUIRED`][required] でアノテーションされてはならない
- フィールドは**しなければならない**（must）[`google.api.field_info.format = UUID4`][uuid4] でアノテーションされている
- フィールド名は**しなければならない**（must）対象メソッドの [`google.api.Publishing.method_settings`][apf] 内の [`google.api.MethodSettings.auto_populated_fields`][apf] エントリにリストされている

### Expected Generator and Client Library Behavior

前述の要件が特定のフィールドで満たされている場合、client libraryジェネレータは**しなければならない**（must）生成されたクライアントで当該フィールドの自動設定を有効にする。

フィールドは、以下の条件のいずれかが成立する場合にのみ自動的に設定され**なければならない**（must）：

- フィールドが明示的な存在をサポートしており、ユーザーによって設定されて **いない**
- フィールドが明示的な存在をサポートしておらず、その値が空文字列である（つまりデフォルト値）

フィールドが `auto_populated_fields` に指定されているが、構造的要件を満たしていない場合、client libraryジェネレータは**してはならない**（must not）そのフィールドの自動設定を有効にする。client libraryジェネレータは**してもよい**（may）生成中にエラーを出力する。

client libraryは**しなければならない**（must）同じリクエストのリトライに対して自動的に設定された値を再利用する。言い換えれば、自動的に設定されたフィールドは、同じリクエストメッセージでの各RPC試行ごとに再生成しては**ならない**（must not）。
