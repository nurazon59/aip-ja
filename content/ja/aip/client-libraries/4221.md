# Client-side retry

多くのAPIには、クライアントがリトライすべきエラーモードが存在する。これらのエラーモードはAPI間で異なるため、ユーザーは利用可能なドキュメントに基づいてclient libraryの周りに手動でリトライロジックを記述することになる。client libraryは、よく知られたエラーモードの自動的なクライアントサイドリトライを実装することで支援する機会がある。

## Guidance

client libraryは**すべきである**（should）自動的なクライアントサイドリトライロジックを提供する。自動的なクライアントサイドリトライロジックを持つclient libraryは**すべきである**（should）リトライ対象のエラーコードとリトライの遅延をユーザーが指定するための仕組みを提供する。

リモート解決されるクライアントリトライ設定をサポートするAPIシステム向けのclient libraryは**すべきである**（should）リモート解決された設定を尊重する。ただし、ユーザーの設定は尊重**しなければならない**（must）。

この機能を実装するclient libraryジェネレータは**しなければならない**（must）リトライ設定を受け入れる。このリトライ設定は**してもよい**（may）`protoc` プラグインオプションを介して提供される。リトライ設定が与えられていない場合、client libraryジェネレータは**すべきではない**（should not）デフォルトのリトライ設定を生成する。

### Retry implementation

client libraryは**すべきである**（should）クライアントサイドリトライをユーザーに対して透過的にする。ユーザーは**すべきではない**（should not）明示的にクライアントサイドリトライにオプトインする必要があるが、ユーザーはクライアントサイドリトライを完全に無効化する方法を持た**しなければならない**（must）。

### Retry configuration mechanisms

client libraryは**すべきである**（should）ユーザーがクライアントサイドリトライ設定を制御できる仕組み（クライアントサイドリトライの完全な無効化を含む）を公開する。

例えば、gRPCサービスのGo client libraryでは、クライアント初期化時に [WithDisableRetry][0] オプションを指定して、自動クライアントサイドリトライロジックの使用を無効化できる。

```go
opts := []grpc.DialOption{
  grpc.WithDisableRetry(),
  grpc.WithTransportCredentials(creds),
}

cc, err := grpc.Dial("my.api.net:443", opts...)
if err != nil {
  // ...
}
```

### Remotely-resolved client configuration

一部のAPIシステムには、クライアントサイドリトライ設定を含むリモート定義の設定をクライアントが取得するための組み込みメカニズムがある。

例えば、gRPCは[自動クライアントサイドリトライ][2]の設定を含む[クライアント設定の解決][1]をサポートしている。

client libraryは**すべきである**（should）リモート解決された設定を尊重する。ただし、ユーザーが前述のclient libraryの[リトライ設定メカニズム](#retry-configuration-mechanisms)を介してそれを上書きする場合を除く。

### Client library generator retry configuration

クライアントサイドリトライを実装するclient libraryジェネレータは**しなければならない**（must）リトライ設定を受け入れる。これは、APIプロデューサーが自サービスに最適なリトライ設定を提供できるようにするためである。

例えば、gRPCクライアントサイドリトライは、[gRPC Service Config][4] 内の [RetryPolicy][3] で設定される。以下は、`google.example.library.v1.LibraryService` サービスのすべてのメソッドに `RetryPolicy` を適用し、明示的に名前指定されたメソッドには `RetryPolicy` を適用しない設定である。

<!-- prettier-ignore-start -->
```json
{
  "methodConfig": [{
    "name": [{ "service": "google.example.library.v1.LibraryService" }],
    "waitForReady": true,
    "timeout": "60s",
    "retryPolicy": {
      "maxAttempts": 3,
      "initialBackoff": "1s",
      "maxBackoff": "10s",
      "backoffMultiplier": 2,
      "retryableStatusCodes": ["UNAVAILABLE"]
    }
  },
  {
    "name": [
      { "service": "google.example.library.v1.LibraryService", "method": "CreatePublisher" },
      { "service": "google.example.library.v1.LibraryService", "method": "DeletePublisher" },
      { "service": "google.example.library.v1.LibraryService", "method": "CreateBook" },
      { "service": "google.example.library.v1.LibraryService", "method": "DeleteBook" },
      { "service": "google.example.library.v1.LibraryService", "method": "UpdateBook" },
      { "service": "google.example.library.v1.LibraryService", "method": "MoveBook" }
    ],
    "waitForReady": true,
    "timeout": "10s"
  }]
}
```
<!-- prettier-ignore-end -->

リトライ設定は**してもよい**（may）`protoc` プラグインオプションである。例えば、ジェネレータは次のようなオプションで設定ファイルのパスを受け入れられる：

    --{plugin_name}_opt="retry-config=/path/to/config.file"

リトライ設定がない場合、ジェネレータは**すべきではない**（should not）「デフォルト」のリトライ設定を生成する。これにより、生成されたclient libraryは、ユーザーが設定しない限り何もリトライしない状態になる。
