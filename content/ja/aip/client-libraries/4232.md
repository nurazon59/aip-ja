# Method signatures

Protocol BufferのRPCでは、各RPCは正確に1つの引数（メッセージ）を受け取る。しかし、非常にシンプルなリクエストの場合、完全なメッセージ構造を送信するのは面倒である。

多くのRPCは、リクエストのどの部分が重要で一般的に使用されるかについての情報を提供する。多くの言語では、関数やメソッドは複数の位置引数やキーワード引数を受け取る。

## Guidance

一部のAPIは、単一のリクエストオブジェクトから個別の引数に効果的に変換する方法を示すアノテーションを提供しており、client libraryは**してもよい**（may）これらのヒントに基づいてオーバーロードを提供する。

ただし、この機能を実装するclient libraryは**しなければならない**（must）完全なリクエストオブジェクトを受け入れるデフォルトの動作を保持する。言い換えれば、APIが _既に_ 公開済みのAPIに _このアノテーションを追加_ した場合、結果として生じるライブラリの変更は後方互換性が**なければならない**（must）。

client libraryジェネレータは**してもよい**（may）環境に応じて、一部のケースではこの機能を提供し、他のケースでは提供しないことを選択する。例えば、以下の戦略はいずれも許容される：

- シグネチャ内のすべての引数がプリミティブ型である場合に限りオーバーロードを提供する。
- 複数のシグネチャを提供すると競合が生じる場合、最初のシグネチャのみにオーバーロードを提供する。
- 上記の任意の組み合わせ。

これらのすべての状況において、リクエストオブジェクトが常に受け入れられるという要件は依然として適用される。

さらに、client libraryジェネレータは**してもよい**（may）RPCタイプのサブセットに対してこの機能をサポートすることを選択する。対象は以下の通り：

- Unary
- Server Streaming
- Client Streaming
- Bi-directional Streaming

### Method Signatures

[`google.api.method_signature`][method_signature] アノテーションが付与されたRPCは、サポートされている場合にフラット化されたメソッドシグネチャを持つオーバーロードが望まれることを示す。文字列にはカンマ区切りの引数が順に含まれる。フィールド名にピリオド（`.`）文字が含まれている場合、これはネストされたフィールドを示す。

RPCはこのアノテーションを複数回提供して、複数のシグネチャを指定できる。ここでは順序が重要である：状況によっては、提供されたすべてのシグネチャに対してオーバーロードを生成できない場合がある。この状況では、client libraryジェネレータは**しなければならない**（must）「first match wins」戦略に従う（競合セット内の最初のシグネチャのオーバーロードを生成し、残りを破棄する）。

**Note:** この帰結として、メソッドシグネチャアノテーションの _追加_ のみが後方互換性のある変更であることが保証される。

### Required Arguments

多くの場合、[AIP-203][] で説明されているように、リクエストメッセージの特定のフィールドは一貫してrequiredである。

client libraryは通常これに関するバリデーションを実行すべきではない（それはサーバーの役割である）が、client libraryは**してもよい**（may）言語に適切であれば、メソッドシグネチャ内のrequired引数とoptional引数を区別する。この目的において、フィールドは `google.api.field_behavior` アノテーションの値が `REQUIRED` である場合にrequiredと見なされる：

```proto
message TranslateTextRequest {
  // The text to translate.
  string q = 1 [(google.api.field_behavior) = REQUIRED];
}
```

**Note:** フィールド動作のアノテーションは _フィールド_ に付与され、メソッドには付与されない。

### Restrictions

RPCが [`google.api.method_signature`][method_signature] にネストされたフィールド（例：`"foo.bar.baz"`）をリストしている場合、個々のコンポーネントフィールドのうち最後のものを除いて繰り返し（repeated）であってはならない（例を続けると、`baz` はrepeatedであり得るが、`foo` や `bar` はrepeatedであってはならない）。この機能を実装するコードジェネレータは**しなければならない**（must）非終端のrepeatedフィールドをフィールド名として検出した場合、説明的なエラーメッセージとともにエラーを出力する。

required引数がある場合、すべてのrequired引数はoptional引数の前に現れることが期待される。この機能を実装するコードジェネレータは**すべきである**（should）optional引数の後にrequiredフィールドを検出した場合、説明的なエラーメッセージとともにエラーを出力し、結果として生成されるclient libraryが有効でない場合は**しなければならない**（must）エラーを出力する。

### Compatibility

既存の `google.api.method_signature` エントリの削除、並べ替え、または変更は、client libraryにとって破壊的変更である。関連する生成メソッドは完全に削除されるか、そのシグネチャが変更され、どちらもユーザーコードを壊す。
