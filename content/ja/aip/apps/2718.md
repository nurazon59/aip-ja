# 他のAPIのオブジェクトへの参照

APIが他のAPIのオブジェクトを参照する必要がある場合がある。例えば、Sheetsのセルがその意味を説明するSlidesのデッキを参照したい場合などである。この種の相互参照は、ターゲットAPIで可能な限り自然に使用できるようにすべきである。

## ガイダンス

APIが別のAPI（「ターゲット」API）を介してアクセスされるオブジェクトを参照する場合、APIが提供する参照は、その参照がOne Platformに準拠しているかどうかに依存する。

- フィールドが常に単一の特定のターゲットAPIへの準拠参照を保持する場合、提供する参照は、ターゲットAPIの命名階層のトップレベルからの相対URIで**なければならない**（must）。例えば、オブジェクトの完全なURLが `https://koalas.googleapis.com/koalas/123435` である場合、返す参照は相対URI `koalas/123435` で**しなければならない**（must）。さらに、ドキュメントではその参照が `koalas.googleapis.com` で使用されることを明確にしなければならない。

  これにより、開発者は返されたパスを直接使用して、自然な方法で参照を使用できる。また、どのAPIがその参照を理解するかを開発者が把握できる。

- フィールドが常に準拠参照を保持するが、単一のターゲットAPIに限定されない場合、提供する参照は、特定のターゲットAPIのパスを含む相対URI（例：`koalas.googleapis.com/koalas/123435`）で**なければならない**（must）。これにより、開発者は形式化された使用可能な方法で正しいターゲットを得られる。

- それ以外の場合、返す参照はターゲットAPIで可能な限り自然に使用できるように選択**されなければならない**（must）、APIとそのバージョンを文書化しなければならない。例えば、ApiaryベースのKoala APIが `GetKoalaData` メソッドでIDを使用する場合、パラメータとして使用できるIDを返さ**なければならない**（must）とともに、Koala API 1.0で動作するIDを使用していることを文書化する。

### 命名

準拠参照の場合、フィールド名に「reference」や「name」という用語を使用することを避ける**べきである**（should）。すなわち、`koala_reference` や `koala_name` よりも `koala` が推奨される（[AIP-122][]参照）。

その他のターゲットAPIの場合、明確性のためにそのAPIの用語を名前に反映させる**べきである**（should）。例えば、ターゲットAPIの `GetKoalaDataRequest` に `koala_num` フィールドがある場合、フィールド名はおそらく `koala_num` にする**べきであり**（should）、または `responsible_koala_num` のように名前に `koala_num` を含める。
