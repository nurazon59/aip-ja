# プロジェクトとアプリ

## ガイダンス

### リソース階層

リソース名の一部としてApp IDを利用するAPIは、親にProject識別子を含める**しなければならない**（must）：

```
apps/*/items
```

代わりに以下のように**しなければならない**（must）：

```
projects/*/apps/*/items
```

Project識別子は[Cloudガイダンス][aip-2510]に従わ**なければならない**（must）。App IDはグローバルに一意であるため、Firebase APIは[一意リソースルックアップ][aip-159-unique]パターンを許可し、Project ID/Numberの代わりに `-` を使用できるようにする**すべきである**（should）。例えば：

```
projects/-/apps/1:111222333444:web:abcdef0123456789/things
```

### App IDの参照

SDKまたはAPIのフィールド、メソッド、パラメータでApp IDを参照する場合、プラットフォームとして正しい頭字語 `app_id` を使用**しなければならない**（must）（`app`、`application_id`、`gmp_app_id` などではなく）。
