# BIIスキーマの原則

Built-in intent（BII）により、サードパーティの開発者は自身のActionが何をできるかを宣言できる。開発者がBIIにActionを登録すると、Googleはそのbuilt-in intentにマッチするユーザーリクエストを満たすためにそのActionを呼び出せる。

## 用語

以下の一般的な原則では、次の用語を使用する：

- **Built-in intent（BII）**: ユーザークエリに応答してActionを実行するために互換性のあるAPIを呼び出すために使用される履行スキーマ。Built-in intentを使用すると、サービスは自身の履行能力をGoogleに表明できる。Built-in intentに登録し、intentパラメータを履行にマッピングすることで、Google Assistantが自然言語クエリに応答してタスクを実行するためにサービスを呼び出すことが可能になる。

  BIIは、操作（完全なリストは後述）とSchema.orgエンティティ（別名タイプ）で構成される。例えば、`CREATE_RESERVATION` は新しい予約を開始するためのBIIであり、このintentは航空、ホテル、レストラン、その他の予約など様々なドメインに適用される。

- **操作**: タイプをラップするアクション。
- **タイプ**: 操作の対象となるオブジェクト。

## 一般原則

各BIIはAPIのように扱われる**べきである**（should）。BIIは技術的には履行スキーマであるが、サードパーティの開発者にActionが処理するリクエストの構造を伝えるという点でAPIと同じ役割を果たす。

このAIPは、スキーマ作成の原則を操作固有およびタイプ固有の情報とともに詳述し、その後BIIの操作一覧を示す。

### スキーマ作成

- 必要なセマンティクスが既に存在する場合は、既存のSchema.orgのタイプとプロパティを再利用する**べきである**（should）。
- BIIの命名式は以下の通りで**なければならない**（must）：
  - **操作 + タイプ** `:=` `VERB` ["Operation" from list below] + '\_' + `NOUN` ["Type" from Schema.org]。
  - タイプは複数のトークンを持つ複合名詞であり得る。複数のトークンを持つタイプは '\_' で区切られる。
    - _例: `CREATE` + '\_' + `MONEY_TRANSFER` = `CREATE_MONEY_TRANSFER`_
- BII名はCAPITAL_SNAKE_CASEで**なければならない**（must）（例: VERB_NOUN_NOUN）。
- 詳細については、[こちら](https://schema.org/docs/documents.html)を参照。

#### 操作

- 操作は、単純で一般化可能な動詞の小さなカタログで構成される。これらはAIPの下部「List of BII Operations」に記載されている。
- いかなる操作のカバーする範囲にも重複があっては**ならない**（must not）（相互排他性）。
  - _例: `GET` と `CREATE` は範囲が重複しない。どちらもオブジェクトを提示することを含むが、`GET` は既存のオブジェクトを取得するためのものであり、`CREATE` はまだオブジェクトが存在せず、作成することを前提としている。_
- 操作とタイプの組み合わせは意味的に理にかなっている**べきである**（should）。選択したタイプで動作する操作がない場合：
  - Actionが意味的に妥当な結果を得るために既存の操作にアタッチできる、意味的範囲内の新しいタイプを選択する。
    - _例: 予約のチェックインを満たすリクエストがあるとする。`CREATE_CHECKIN` はほとんどの意味的に理にかなっていない。何も作成されていないからである。さらに検討した結果、別の操作を選び、BIIを `UPDATE_RESERVATION` とする。これは予約のチェックインのセマンティクスとより整合する。_
  - または、新しい操作を作成する。この手順は、スキーマに要求されるActionをカバーできる操作がリストにない場合にのみ実行する。
    - _例: 進行中のアクティビティを制御するための操作として `START`、`STOP`、`RESUME` しかないとする。メディア再生の進行中のアクティビティを一時的に停止する（終了はしない）履行リクエストが来る。`STOP_MEDIA` というBIIを作成しようとするが、この操作はAPIがアクティビティを終了することを意味することに気づく。そこで新しい操作 `PAUSE` を作成する。これは「アクティビティを一時的に停止/一時中断する」として動作し、既存の操作 `STOP` と範囲が重複しない。新しいBIIは `PAUSE_MEDIA` となる。_

#### タイプ

- タイプはSchema.orgのタイプとして定義され**なければならない**（must）。
- タイプは操作が実行される_オブジェクト_で**なければならない**（must）。
- タイプの意味的範囲は一般化可能である**べきであり**（should）、具体性はプロパティに持たせても**よい**（may）。
  - _例: [Reservation][] はタイプであり、`provider` というプロパティを持つ。`provider` プロパティはサービスプロバイダ、サービス運営者、またはサービス提供者を指定する。これにより、[Reservation][] タイプを使用して作成されたBIIは任意のサービスプロバイダに一般化可能であり、サービスプロバイダ情報は [Reservation][] のプロパティ `provider` を介して引き続き渡される。_
- 異なるタイプは明確に異なるプロパティを持つ**べきである**（should）。
- BIIは、APIを呼び出す際の意味的曖昧さを避けるために十分に具体的で**なければならない**（must）。つまり、ユーザーが期待していないActionを誤ってトリガーしてはならない。
  - _例: 証券口座とアカウント: 証券口座と指定しない場合、`GET_ACCOUNT` BIIがソーシャルメディアアカウントなど別の種類のアカウント情報を取得するためのActionを満たすAPIを呼び出す可能性がある。`GET_BROKERAGE_ACCOUNT` とすることで、APIが金融口座の正しい範囲を取得することが保証される。_
- 関数のプロトタイプ/パラメータセットは操作にバインドされる**べきである**（should）。つまり、操作がActionの履行範囲を決定する。
- 各BII操作はSchema.orgタイプを取ら**なければならない**（must）。
- BIIスキーマでは、同じSchema.orgタイプでも異なる動詞と組み合わせた場合、異なるプロパティを取っても**よい**（may）。
  - _例: `GET_MESSAGE` と `CREATE_MESSAGE` は [Message][] の異なるが重複するプロパティセットを使用する。_

## ユーザークエリの処理

- オブジェクトレベルでの汎用的/未解決の検索条件は、Descriptionフィールドを通じて渡される**べきである**（should）。
- クエリ引数がタイプのプロパティとして認識できる場合はプロパティとして渡される**べきである**（should）。一方、引数が単にタイプの説明として認識される場合はDescriptionフィールドを通じて渡される**べきである**（should）。
  - _例: "find funky blues songs on YouTube" -> get_media_object(description="funky blues")。_

## BII操作一覧

| 動詞   | 定義                                                    | BIIスキーマの例     |
| ------ | ------------------------------------------------------------- | ------------------ |
| GET    | オブジェクトを取得して提示する。曖昧な場合はオブジェクトのリスト | GET_CALL_HISTORY   |
| CREATE | オブジェクトを作成する                                              | CREATE_ORDER       |
| DELETE | 既存のオブジェクトを削除または終了する                        | DELETE_RESERVATION |
| START  | 進行中のアクティビティを制御する: アクティビティを開始                   | START_GAME         |
| STOP   | 進行中のアクティビティを制御する: アクティビティを停止                    | STOP_ALARM         |
| PAUSE  | 進行中のアクティビティを制御する: アクティビティを一時的に停止/一時中断  | PAUSE_SONG         |
| RESUME | 進行中のアクティビティを制御する: アクティビティを継続                | RESUME_TIMER       |
